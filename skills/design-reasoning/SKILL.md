# 设计推理

## 核心流程

```
需求分析 → 设计约束推理 → 方案生成 → 一致性验证
```

## 工作步骤

### 1. 需求分析
- 提取设计目标：用户角色、核心场景、品牌调性
- 识别约束：平台（Web/Mobile/桌面）、无障碍要求、性能预算

### 2. 设计约束推理
- 色彩推理：品牌色 → 语义色（primary/success/warning/error）→ 中性色阶
- 间距推理：基础单位 → 间距比例（4/8/12/16/24/32/48）
- 字体推理：字号比例 → 行高 → 字重组合
- 组件推理：原子组件 → 分子组件 → 有机组件

### 3. 方案生成
- 输出设计 Token（CSS 变量 / Tailwind 配置）
- 输出组件规范（Props / 状态 / 变体）
- 输出交互规范（动画 / 过渡 / 响应式断点）

### 4. 一致性验证
- 检查色彩对比度（WCAG AA/AAA）
- 检查间距一致性
- 检查组件 Props 完整性

## 输出格式

```markdown
# 设计决策记录
## 设计 Token
[CSS 变量 / Tailwind 配置]
## 组件规范
[组件列表与 Props]
## 交互规范
[动画与响应式规则]
```

# 评估驱动开发

## 核心流程

```
定义指标 → 建立基准 → 实现方案 → 盲比较 → 数据决策
```

## 工作步骤

### 1. 定义评估指标
- 性能指标：延迟、吞吐、内存、CPU
- 质量指标：代码行数、圈复杂度、类型覆盖率
- 用户体验指标：首屏时间、交互延迟、错误率

### 2. 建立基准
- 记录当前实现的基准数据
- 确保测试环境一致（相同数据量、相同硬件）

### 3. 盲比较
- 隐藏方案标识，避免确认偏差
- 多次运行取中位数，消除随机波动
- 记录原始数据，不丢弃异常值

### 4. 数据决策
- 输出对比表格（方案 × 指标）
- 标注统计显著性
- 推荐最优方案并说明理由

## 约束

- 禁止无基准数据的"优化"
- 禁止挑选有利数据的确认偏差
- 每次比较至少 3 次运行取中位数

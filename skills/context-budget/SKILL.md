# 上下文预算管理

## @Examples

```
用户: "上下文太长了"
Claude: /context-budget → 压缩历史 → 摘要关键决策 → 保留最近状态

用户: "怎么管理上下文？"
Claude: /context-budget → 70%阈值 → 主动压缩策略
```

## Token 预算规则

| 阈值  | 动作                       |
| ----- | -------------------------- |
| > 50% | 警告，考虑压缩             |
| > 70% | 主动压缩历史消息           |
| > 85% | 激进压缩，只保留关键上下文 |

## 压缩策略

### 1. 消息摘要

- 保留：关键决策、用户偏好、当前任务状态
- 丢弃：重复讨论、已验证结论、临时调试信息

### 2. 固化到文件

- 项目上下文 → `specs/TASK_NAME.md`
- 用户偏好 → `experiences/`
- 架构决策 → `docs/architecture.md`

### 3. 清理时机

- 任务完成后立即压缩
- 切换项目时压缩
- 上下文超 70% 时立即压缩

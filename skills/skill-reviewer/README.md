## Overview

skill-reviewer是一个专为skill质量审查而设计的元技能，它能够基于华为研发规范和技术最佳实践，对skill进行全面、系统性的质量评估和深度分析。当用户提示需要审查、审核或检查skill时，调用本skill。

## Features

### ✅ 审查维度

#### 合规性审查
- 命名规范检查（G.NAM.01-G.NAM.04）- 检查烤串命名法、功能性命名模式及避免语义噪音等规范
- 目录结构检查（G.FMT.01-G.FMT.02）- 验证Skill名称与目录一致性、脚本目录规范性
- 文件格式检查（G.FMT.04-G.FMT.06）- 验证变量格式、命名语言要求等文件规范
- 内容要求检查（G.EXP.01-G.EXP.06）- 确保有效描述、术语一致性、输入输出明确性

#### 实践审查
- 文档完整性与结构化检查
- 错误处理和日志检查

## Prerequisites & Setup
- 目标技能：准备目标技能（即待评审技能），可以是本机任意目录下的skill，不一定已经在Agent安装完成

## Trigger Prompts & User Scenarios
### 触发Agent调用此Skill的用户提示词示例
- **场景1：** 用户需要评审单个skill
- *用户提示词：* “评审某个skill”
- *Agent预期行为：* Agent触发skill-reviewer对用户指定的skill进行评审，并输出评审报告

- **场景2：** 用户需要评审指定路径下的所有skill
- *用户提示词：* “评审某个目录下的skill”
- *Agent预期行为：* Agent触发skill-reviewer对用户指定的skill进行评审，并输出评审报告

### 使用时机建议

- **开发过程中**: 每完成一个功能模块后进行初步审查
- **代码提交前**: 确保skill质量达到团队标准
- **版本发布前**: 进行全面的质量扫描
- **定期审查**: 每周或每月进行定期的skill质量评估

## Input & Output Specification

### 审查报告结构

```markdown
## Skill评审报告
### 1. 概览
- 检查项总数: 25
- 通过检查项: 22
- 警告项: 2 
- 失败项: 1
- 总体评分: 88/100
- 状态: pass_with_warnings

### 2. 分类详情
#### 命名规范检查 (G.NAM.*)
- 烤串命名法检查: 通过
- 功能性命名模式检查: 通过
- 避免语义噪音检查: 通过
- 子分类后缀检查: 通过

#### 目录结构检查 (G.FMT.*)
- 目录与skill名称一致性检查: 通过
- 脚本目录检查: 警告

#### 文件格式检查 (G.FMT.*)
- 变量使用格式检查: 通过
- 命名语言要求检查: 通过

#### 内容要求检查 (G.EXP.*)
- 有效描述检查: 通过
- 术语一致性检查: 通过
- 输入输出前置条件明确性检查: 失败
- 其他检查项: 通过

## 3. 详细问题列表
[详细问题描述里带有文件路径和行号]

## 4. 修复建议
[带有优先级的修复建议]
```

### 检查结果类型

- **通过 (pass)**: 符合规范要求
- **警告 (warning)**: 基本符合但建议改进
- **失败 (fail)**: 不符合规范要求，必须修复

### 总体评分标准

- **95-100分**: 优秀，完全符合所有规范
- **85-94分**: 良好，有小幅改进建议
- **75-84分**: 合格，部分项目需要改进
- **60-74分**: 待改进，存在多项需要修复的问题
- **60分以下**: 不合格，严重不符合规范要求

## Limitations & Known Issues

- 仅支持基于华为《Agent Skill开发规范》的检查
- 为确保本skill的使用效果，推荐使用CodeAgent+MiniMax-M2.5
- 如果使用环境中已经存在类似功能的skill，建议显性化调度本skill

## Non-Functional Metrics

- **平均执行时长:** 取决于检查的Skill规模，通常1-3分钟
- **Token消耗:** 每次检查约消耗15-20K tokens
- **准确率:** 规范符合性检查准确率达到100%

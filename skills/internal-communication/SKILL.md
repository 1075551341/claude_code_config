# 内部沟通

## 核心能力

**编写内部邮件、团队公告、项目通知、跨部门沟通文档。**

---

## 适用场景

- 内部邮件编写
- 团队公告
- 项目更新通知
- 跨部门沟通

---

## 沟通原则

### CLEAR 原则

```
C - Concise 简洁：直入主题
L - Logical 逻辑：结构清晰
E - Explicit 明确：具体行动
A - Appropriate 适当：受众匹配
R - Respectful 尊重：专业礼貌
```

### 沟通矩阵

| 类型 | 受众 | 渠道 | 频率 |
|------|------|------|------|
| 紧急 | 全员 | IM/电话 | 即时 |
| 重要 | 相关人员 | 邮件 | 当天 |
| 常规 | 团队 | 周报 | 每周 |
| 参考 | 全员 | Wiki | 按需 |

## 邮件模板

### 项目更新

```markdown
主题：【项目更新】XXX项目进度汇报

各位同事好，

## 项目进展
- 完成情况：已完成80%
- 本周成果：
  1. 完成核心功能开发
  2. 通过第一轮测试
  3. ...

## 下周计划
- 完成剩余功能
- 准备上线部署

## 风险提示
- 需要协调测试资源
- ...

## 需要支持
- 请各部门在周五前确认...

如有问题请随时沟通。

---
张三
项目组
```


## 沟通技巧

### 坏消息传达

```markdown
结构：
1. 直接说明问题
2. 解释原因（简短）
3. 提供解决方案
4. 表达支持

示例：
项目需要延期两周。

原因是关键资源临时调整。

我们已制定追赶计划，增加人力投入。

如有影响请告知，我们将协调解决。
```

### 好消息传达

```markdown
结构：
1. 开门见山
2. 归功团队
3. 展望未来

示例：
很高兴通知大家，项目按时交付！

感谢团队的努力和各部门的支持。

下阶段我们将继续...
```

## 注意事项

```
必须：
- 主题清晰
- 分点陈述
- 明确行动项
- 检查收件人

避免：
- 过长邮件
- 模糊表述
- 抄送无关人员
- 情绪化表达
```

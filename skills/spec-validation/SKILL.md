# 规格验证

## 触发条件
- spec.md 文件创建或更新后
- 用户要求验证规格
- 实现完成后对照规格验收

## 执行步骤

1. **解析规格**：读取 spec.md，提取业务规则和验收条件
2. **生成验证清单**：将每个验收条件转为可执行检查项
3. **执行验证**：逐项检查实现是否满足规格
4. **输出报告**：通过/失败/待验证，附证据

## 验证格式
- 每条规则使用 EARS 格式：When [event], the [system] shall [response]
- 验收条件：触发场景 → 预期行为

## 验收标准
- 所有业务规则有对应验证项
- 验证结果有证据支撑
- 未通过项有明确原因

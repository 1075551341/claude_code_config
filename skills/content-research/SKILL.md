## 文章提取流程

### 1. URL 获取

```
使用工具：
- WebFetch 工具
- Readability 类库
- newspaper3k (Python)
```

### 2. 内容解析

```python
# Python 示例
import newspaper

article = newspaper.Article(url)
article.download()
article.parse()

title = article.title
text = article.text
authors = article.authors
publish_date = article.publish_date
```

### 3. 结构化输出

```markdown
## 文章信息
- 标题：{title}
- 作者：{authors}
- 发布：{date}
- 来源：{url}

## 内容摘要
[300字核心内容]

## 关键观点
1. {point_1}
2. {point_2}
3. {point_3}

## 素材引用
[可直接引用的段落]
```

## 素材收集方法

### 来源优先级

| 来源 | 可信度 | 用途 |
|------|--------|------|
| 官方文档 | 高 | 技术引用 |
| 学术论文 | 高 | 深度分析 |
| 行业报告 | 中 | 趋势参考 |
| 技术博客 | 中 | 实践案例 |
| 社区讨论 | 低 | 补充视角 |

### 收集模板

```markdown
## 素材卡片

### 来源
- 标题：{title}
- 链接：{url}
- 类型：{type}

### 内容
[关键内容摘要]

### 价值
- 可用于：{use_case}
- 关键词：{keywords}

### 引用格式
> [引用内容]
```

## 内容分析框架

### 观点提取

```
分析方法：
1. 识别核心论点
2. 梳理支持证据
3. 标注逻辑结构
4. 评估可信度
```

### 知识图谱

```markdown
主题 → 子主题 → 观点 → 证据

示例：
AI开发 → Prompt工程 → Few-shot → 实验数据
                      → Chain of Thought → 论文引用
```

## 写作研究支持

### 研究清单

```markdown
研究前：
- 明确写作主题
- 确定信息边界
- 列出关键词

研究中：
- 收集多源素材
- 标注来源信息
- 提取关键观点
- 记录矛盾观点

研究后：
- 整理素材分类
- 撰写核心摘要
- 建立引用索引
- 验证关键事实
```

### 输出格式

| 格式 | 内容 |
|------|------|
| 素材库 | Markdown 素材卡片集合 |
| 摘要集 | 各来源核心内容摘要 |
| 观点表 | 多观点对比表格 |
| 引用集 | 可直接引用的段落 |

---

## 注意事项

```
必须：
- 标注来源信息
- 保持原文准确性
- 区分事实与观点
- 处理版权问题

避免：
- 单一来源依赖
- 歪曲原意
- 遗漏关键信息
- 未验证引用
```

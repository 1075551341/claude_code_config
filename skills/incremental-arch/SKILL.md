# 增量架构

## 核心流程

```
变更检测 → AST分块 → 差量计算 → 增量同步
```

## 工作步骤

### 1. 变更检测
- 监听文件变更事件
- 计算 Merkle 哈希判断是否真正变更
- 跳过格式化等无语义变更

### 2. AST 感知分块
- 按语法结构（函数/类/模块）分块，非按行数
- 保留语义完整性（不拆半函数）
- 每块附带依赖关系（import/调用）

### 3. 差量计算
- 对比前后 AST，识别增/删/改
- 生成最小差量包
- 标记影响范围（哪些模块受影响）

### 4. 增量同步
- 仅同步变更块，非全量
- 更新依赖图
- 触发受影响模块的重新验证

## 约束

- 分块必须保持语义完整
- 差量包必须可逆（支持回滚）
- 同步顺序遵循依赖拓扑

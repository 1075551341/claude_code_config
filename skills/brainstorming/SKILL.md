# 设计头脑风暴

## @Examples

```
用户: "我们新功能要用什么架构？"
Claude: /brainstorming → 决策矩阵 → 推荐方案

用户: "帮我设计用户模块"
Claude: /brainstorming → 三方案对比 → 决策点确认
```

## 核心流程

```
💡 发散思考 → 🎯 聚焦收敛 → ✅ 验证确认
```

## 工作步骤

### 1. 问题定义

```markdown
## 我们在解决什么问题？
- 问题陈述：
- 影响范围：
- 成功标准：

## 约束条件
- 时间约束：
- 技术约束：
- 资源约束：
```

### 2. 方案发散

```markdown
## 方案一：[名称]
- 核心思路：
- 优点：
- 缺点：
- 复杂度：⭐⭐⭐

## 方案二：[名称]
- 核心思路：
- 优点：
- 缺点：
- 复杂度：⭐⭐

## 方案三：[名称]
...
```

### 3. 决策矩阵

```markdown
| 维度 | 权重 | 方案A | 方案B | 方案C |
|------|------|-------|-------|-------|
| 开发成本 | 30% | 8 | 6 | 9 |
| 性能 | 25% | 7 | 9 | 6 |
| 可维护性 | 25% | 9 | 7 | 5 |
| 扩展性 | 20% | 6 | 8 | 7 |
| **总分** | 100% | **7.55** | **7.45** | **6.85** |
```

### 4. 风险评估

```markdown
## 推荐方案风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 风险A | 中 | 高 | 预留备份方案 |
| 风险B | 低 | 中 | 监控告警 |

## 决策点
- [ ] 风险可接受
- [ ] 缓解措施可行
- [ ] 有回滚方案
```

### 5. 验证清单

```markdown
## 方案验证
- [ ] 符合业务需求
- [ ] 技术可行性确认
- [ ] 资源评估完成
- [ ] 时间线合理
- [ ] 风险可控

## 下一步
1. 编写详细设计文档
2. 创建任务分解
3. 开始实施
```

## 提问框架

### 需求澄清

```markdown
- 这个功能的用户是谁？
- 使用频率如何？
- 最核心的场景是什么？
- 有没有参考案例？
```

### 技术深入

```markdown
- 数据量级预估？
- 并发要求？
- 一致性要求？
- 是否需要灰度发布？
```

### 边界确认

```markdown
- 错误情况如何处理？
- 超时怎么处理？
- 数据怎么清理？
- 是否需要审计日志？
```

## 输出格式

```markdown
# 设计决策记录

## 背景
[问题背景]

## 决策
[最终方案]

## 理由
[为什么选择这个方案]

## 替代方案
[考虑过但未选择的方案]

## 后果
[这个决策的影响]
```

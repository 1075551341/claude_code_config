# 请求代码审查

## 核心原则

**使用精确构建的上下文派发代码审查子代理——绝不是你的会话历史。**

## 必须审查的时机

- 子代理驱动开发中每个任务完成后
- 完成主要功能后
- 合并到主分支前

## 审查流程

### 步骤 1: 获取提交范围

```bash
# 获取 BASE_SHA 和 HEAD_SHA
git merge-base origin/main HEAD  # BASE_SHA
git rev-parse HEAD               # HEAD_SHA

# 或简化
BASE_SHA=$(git merge-base origin/main HEAD)
HEAD_SHA=$(git rev-parse HEAD)
```

### 步骤 2: 派发审查代理

使用 Task 工具派发 `code-reviewer` 代理。

### 步骤 3: 填写审查模板

```markdown
## 审查请求

### 实现内容
[简要描述实现了什么功能]

### 计划/需求
[链接到计划文档或需求说明]

### 提交范围
- BASE_SHA: [基准提交]
- HEAD_SHA: [当前提交]

### 描述
[详细说明变更内容和原因]

### 关键文件
- `path/to/file1.ts` - [变更说明]
- `path/to/file2.ts` - [变更说明]

### 需要关注的问题
- [可能需要审查者特别注意的点]
```

### 步骤 4: 处理反馈

| 严重级别 | 处理方式 |
|----------|----------|
| Critical（关键） | 立即修复 |
| Important（重要） | 继续前修复 |
| Minor（次要） | 记录后续处理 |

## 审查请求模板

```markdown
# 代码审查请求

## 基本信息
- **PR/分支**: #123 / feature/user-auth
- **审查范围**: 用户认证功能实现
- **审查者**: @reviewer

## 变更概述

### 新增功能
- 用户登录/注册
- JWT Token 管理
- 权限验证中间件

### 修改文件
| 文件 | 变更类型 | 说明 |
|------|----------|------|
| src/auth/service.ts | 新增 | 认证服务核心逻辑 |
| src/auth/middleware.ts | 新增 | JWT 验证中间件 |
| src/user/model.ts | 修改 | 添加密码字段 |

### 提交历史
```
abc1234 feat: 添加用户登录功能
def5678 feat: 实现 JWT 验证
ghi9012 fix: 修复 Token 过期处理
```

## 需要关注的点

### 安全相关
- [ ] 密码存储使用 bcrypt
- [ ] JWT Secret 从环境变量读取
- [ ] 敏感接口有权限验证

### 性能相关
- [ ] 数据库查询已优化
- [ ] 避免了 N+1 查询

### 测试覆盖
- [ ] 单元测试覆盖率: 85%
- [ ] 集成测试: 已覆盖主要场景

## 验证步骤

1. 运行测试: `npm test`
2. 检查 lint: `npm run lint`
3. 本地验证: `npm run dev`

## 相关文档
- 设计文档: docs/auth-design.md
- API 文档: docs/api/auth.md
```

## 红旗警告

```markdown
🚨 以下行为会导致审查失败：

- "这太简单了，跳过审查"
  → 简单的代码也有 bug

- 忽略 Critical 级别问题
  → 必须修复才能继续

- 带着 Important 问题继续
  → 技术债务会累积

- 不提供上下文
  → 审查者无法有效评审
```

## 最佳实践

### 提供充分上下文

```markdown
✅ 好的请求：
"实现了用户认证功能，包括登录、注册和 Token 管理。
主要变更在 src/auth/ 目录，请重点关注：
- 密码加密是否安全
- Token 验证逻辑是否完整
- 是否有潜在的安全漏洞"

❌ 不好的请求：
"帮我看看代码"
```

### 明确审查重点

```markdown
✅ 明确指出：
- "请重点审查安全相关代码"
- "数据库查询是否可以优化"
- "错误处理是否完善"

❌ 模糊不清：
- "随便看看"
- "有什么问题吗"
```

### 准备好回应

```markdown
审查反馈后：
1. 逐条阅读反馈
2. 理解问题本质
3. 修复并验证
4. 重新请求审查
```

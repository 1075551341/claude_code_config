# Skill名称: frontend-design-pattern-applier

[TOC]

## 概述

识别前端代码中的僵化、高耦合的面向过程逻辑，通过智能分析推荐最合适的设计模式，并辅助用户完成重构。支持 TypeScript/JavaScript 前端代码的模式识别、重构建议和代码生成。

## 功能特性

* **智能代码坏味道检测**: 自动识别复杂条件分支、散落的对象创建逻辑、代码重复、高耦合等面向过程设计问题
* **设计模式推荐**: 根据具体代码场景智能推荐最合适的设计模式（如策略模式、工厂模式、观察者模式、建造者模式等）
* **重构前后对比**: 生成结构化的分析报告和重构前后代码对比，方便用户理解改进点
* **反过度设计保护**: 强制执行文件数量上限（≤3个新文件），避免过度重构
* **交互式确认**: 在生成重构方案前提供详细的预评估，允许用户调整或取消重构

## 前置条件与配

* **环境变量:** 无需特殊环境变量配置
* **本地工具:** 无需本地工具依赖，Skill 由 AI 助手直接执行

## 输入与输出规范

**输入**

Agent 可接收以下输入：
- **代码文件路径**: 最多 5 个 TypeScript/JavaScript 文件路径
- **代码片段**: 直接提供的代码文本
- **重构目标**: 用户指定的重构方向（可选）

**输出 **

Skill 按工作流程顺序输出以下内容：

1. **代码坏味道分析报告** - 包含问题严重程度、检测到的代码坏味道列表、推荐的设计模式、推荐理由、过度设计检查、文件数量预评估
2. **重构计划** - 包含文件结构规划（优先单文件内部重构）和重构计划表格（新建/修改/删除文件）
3. **重构前后代码对比** - 原始代码与重构后代码并排展示，以及改进点列表
4. **重构方案确认** - 包含推荐设计模式、新建/修改文件数、预计重构复杂度、代码行数变化，以及过度设计检查结果，等待用户确认（确认/调整/取消）

## 限制与已知问题

- **限制 1: 文件数量限制** 仅支持单个模块或最多 5 个文件的分析，禁止整个项目级别分析。超过此限制会导致分析不完整或性能问题。

- **限制 2: 语言支持范围** 仅支持 TypeScript/JavaScript 前端代码。不支持其他编程语言或后端代码的分析。

- **限制 3: 上下文隔离** Skill 在隔离环境中评估代码，可能无法识别某些框架特定的约束或宏定义，建议在重构前充分了解项目上下文。

- **限制 4: 重构阈值** 代码复杂度需达到一定水平才推荐模式重构。对于简单清晰的代码，Skill 可能建议不进行重构以避免过度设计。

- **边界情况: 文件数量约束** 强制执行新建文件数量 ≤3 个的约束。如果重构方案需要创建超过 3 个文件，必须重新设计方案，将逻辑合并到现有文件中。

## 非功能性指标

- **平均延迟:** 取决于代码复杂度和文件数量，通常在 10-30 秒之间。包含代码分析、模式推荐和代码生成时间。

- **Token消耗:** 高。标准执行预计消耗 ~2000-3000 prompt tokens 和 ~1500-2500 completion tokens，取决于代码规模和重构复杂度。

- **可靠性:** 代码识别和模式推荐准确率约 90%。复杂场景可能需要人工调整重构方案。

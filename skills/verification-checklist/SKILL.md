# 完成前验证清单

## 核心原则

**任务完成前必须验证，确保功能正确、测试通过、代码质量达标。**

---

## 适用场景

- 功能开发完成
- Bug 修复完成
- 代码提交前
- 分支合并前
- 发布前检查

---

## 验证清单

### 功能验证

```
□ 核心功能正常工作
□ 边界情况已处理
□ 错误处理覆盖完整
□ 用户交互流畅
□ 性能符合预期
```

### 测试验证

```
□ 单元测试通过
□ 集成测试通过
□ 新测试覆盖新功能
□ 无测试回归失败
□ 关键路径手动验证
```

### 代码质量

```
□ 类型检查通过（tsc --noEmit）
□ 代码检查通过（eslint/ruff）
□ 无硬编码敏感数据
□ 注释清晰必要
□ 命名规范一致
```

### 文档验证

```
□ README 已更新（如有变更）
□ API 文档已更新（如有新接口）
□ 配置说明已更新（如有新配置）
□ CHANGELOG 已更新（版本变更）
```

### Git 验证

```
□ 无未提交的临时文件
□ commit message 规范
□ 无无关文件混入
□ 分支状态正确
```

---

## 验证命令模板

### TypeScript 项目

```bash
# 类型检查
npm run typecheck 或 tsc --noEmit

# 代码检查
npm run lint 或 eslint src/

# 单元测试
npm test

# 构建
npm run build
```

### Python 项目

```bash
# 类型检查
mypy src/

# 代码检查
ruff check src/

# 单元测试
pytest

# 格式检查
ruff format --check src/
```

---

## 提交前检查流程

```bash
1. 运行所有测试
2. 运行类型检查
3. 运行代码检查
4. 检查未跟踪文件
5. 检查暂存文件
6. 编写 commit message
7. 创建提交
```

---

## 合并前检查流程

```bash
1. 确认分支基于最新主分支
2. 运行全部测试套件
3. 检查冲突文件
4. 确认无回归问题
5. 请求代码审查
6. 处理审查反馈
7. 合并到目标分支
```

---

## 发布前检查流程

```bash
1. 确认版本号更新
2. 更新 CHANGELOG
3. 运行完整测试套件
4. 构建生产版本
5. 部署到预发布环境
6. 执行烟雾测试
7. 执行正式发布
```

## 常见问题处理

| 问题 | 处理 |
|------|------|
| 测试失败 | 调用 systematic-debugging |
| 类型错误 | 修复类型定义 |
| 检查警告 | 修复或明确豁免 |
| 回归问题 | 回退变更，重新分析 |
| 文档缺失 | 补充必要文档 |

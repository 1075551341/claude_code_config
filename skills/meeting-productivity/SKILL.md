## 会议类型

| 类型 | 时长 | 参与人数 | 目的 |
|------|------|----------|------|
| 站会 | 15分钟 | 5-10人 | 同步进度 |
| 周会 | 30-60分钟 | 团队 | 总结计划 |
| 决策会 | 60-90分钟 | 关键人员 | 做决定 |
| 头脑风暴 | 60分钟 | 相关人员 | 创意收集 |
| 1on1 | 30分钟 | 2人 | 沟通反馈 |

## 会议准备

### 会前清单

```markdown
## 必要性检查
- [ ] 会议是否必要？
- [ ] 是否可以用邮件/文档替代？
- [ ] 是否有明确目标？

- [ ] 确定参会人员
```

## 会议进行

### 主持人职责

```
1. 准时开始
2. 说明会议目的
3. 控制时间节奏
4. 引导讨论方向
5. 确保人人发言
6. 总结并确认行动项
7. 准时结束
```

### 参会者职责

```
1. 准时参会
2. 提前准备
3. 积极参与讨论
4. 简洁表达观点
5. 记录相关事项
```

### 时间控制

```
技巧:
- 设置计时器
- "停车场"记录偏题讨论
- 指定计时员
- 预留总结时间
```

## 会后跟进

### 行动项跟踪

```
1. 会议结束后24小时内发送纪要
2. 明确责任人和截止日期
3. 设置提醒跟进
4. 下次会议回顾完成情况
```

### 会议评估

```
定期评估:
- 会议是否达成目标？
- 时间是否合理？
- 参会人员是否恰当？
- 如何改进？
```

---

## 减少无效会议

```
替代方案:
- 状态更新 → 文档/邮件
- 简单讨论 → IM沟通
- 信息同步 → 看板工具
- 决策确认 → 投票工具

原则:
- 无议程不开会
- 无准备不开会
- 无结论不结束
```

---

## 注意事项

```
必须:
- 准时开始结束
- 有明确议程
- 记录行动项
- 会后发送纪要

避免:
- 没有准备就开会
- 邀请无关人员
- 讨论跑题
- 没有结论就结束
```

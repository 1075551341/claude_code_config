# Karpathy Guidelines

## 四原则

1. **Think Before Coding** — 先陈述假设；有歧义则呈现多种解读；有更简单方案主动指出
2. **Simplicity First** — 能 50 行不写 200 行；禁止推测性通用化
3. **Surgical Changes** — 只改必须改的；匹配现有风格
4. **Goal-Driven Execution** — 弱命令转强声明式标准（见 CLAUDE.md R1–R11）

## 自检

资深工程师会说「过度设计」？→ 重写。

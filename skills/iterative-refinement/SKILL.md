# 迭代精炼机制（强制执行）

> **铁律：稍复杂任务必须走此流程，不可跳过。**
> 判断标准：涉及>3文件 / 新功能开发 / 需求描述<20字 / >2种可行方案 / 涉及用户偏好

## 强制执行流程

```
任何任务
├─ 简单任务(改bug/改配置/加注释) → 直接执行
└─ 稍复杂任务 → 必须走以下流程 ↓

需求 → 草案v1 → 反问确认 → v2 → 实现 → 关联文件同步 → 交叉验证 → 反馈 → v3 → 达成一致
```

## 三阶段细化（必须按序执行）

### 阶段1: 大范围理清方向
- 确定目标、约束、边界
- 排除不可能方案
- **必须输出**: 目标陈述 + 约束列表 + 可行方案枚举

### 阶段2: 逐层分解
- 将大需求拆为子需求
- 每个子需求独立确认
- **必须输出**: 子需求清单 + 依赖关系 + 优先级排序

### 阶段3: 循环拷问细化
- 对每个子需求反问直到具体可执行
- 达成一致后锁定
- **必须输出**: 可执行方案 + 验收标准

## 反问策略（必须执行，不可跳过）

### 5个必问模板
1. **目标确认**："你希望达到什么效果？能举个具体例子吗？"
2. **约束澄清**："有什么限制条件？（预算/时间/技术/兼容性）"
3. **偏好探测**："A方案更X，B方案更Y，你倾向哪个？"
4. **边界划定**："这个功能需要支持到什么程度？哪些情况不需要？"
5. **验收定义**："怎样算做好了？给我一个验收标准。"

### 反问规则
- **最多3轮反问**：超过3轮仍未明确，给出推荐方案并说明理由
- **每轮缩小范围**：每轮反问后选项减半，收敛到具体方案
- **先给草案再问**：不空手问，先给一个初步方案让用户"改"而非"从零想"
- **确认后锁定**：用户确认后不再反复，按确认方案执行

## 自迭代更新

- 每次迭代结果存入 `~/.claude/experiences/`
- 相似需求再次出现时，直接推荐上次最优方案
- 用户否定上次方案时，记录否定原因避免重复

# 迭代精炼机制（强制执行）

> **铁律：非简单任务必须走此流程，不可跳过。**
> 判断标准：涉及>3文件 / 新功能开发 / 需求描述<20字 / >2种可行方案 / 涉及用户偏好

## 流程

```
1. 头脑风暴  →  发散 ≥3 方案 → 六帽评估 → 收敛选最优
2. 迭代精炼  →  草案 v1 → 反问确认 → v2 → 达成一致 → 锁定
3. 计划先行  →  输出分步计划
4. 执行      →  遵循 R10 简洁优先、R11 安全默认
5. 交叉验证  →  全部检查点通过
6. 学习提取  →  提取模式 → 固化为 instinct/skill/rule
```

## 5 个必问模板

1. "你希望什么效果？举个具体例子？"
2. "有什么限制？（预算 / 时间 / 技术 / 兼容性）"
3. "A 更 X，B 更 Y，倾向哪个？"
4. "支持到什么程度？哪些不需要？"
5. "怎样算做好？给验收标准。"

## 规则

- 先给草案再问；明确后每轮选项减半（不明确则继续发散）
- 确认后锁定执行
- 简单任务（改 bug / 改配置 / 加注释 / 格式化）跳过 1-2-6，3-5 不可跳过

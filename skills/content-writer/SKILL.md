# 内容写作

## 写作流程

```
1️⃣ 调研 → 了解主题、收集素材、分析受众
2️⃣ 大纲 → 结构规划、要点提炼
3️⃣ 初稿 → 快速输出、不过分雕琢
4️⃣ 修改 → 润色语言、补充细节
5️⃣ 校对 → 检查错误、优化排版
```

## 写作技巧

### 标题写法
```
✅ 好标题：
- "10 分钟学会 TypeScript 泛型"
- "React 性能优化的 7 个技巧"
- "为什么我放弃了 Redux"

❌ 不好的标题：
- "关于 TypeScript 的一些介绍"
- "React 学习笔记"
```

### 段落原则
```
- 一个段落只讲一个要点
- 段落不宜过长（3-5 句为宜）
- 使用过渡句连接段落
- 技术内容配代码示例
```

### 代码示例
```
✅ 好示例：
- 有注释说明
- 可以直接运行
- 突出核心逻辑
- 命名清晰有意义

❌ 不好的示例：
- 缺少上下文
- 变量名无意义 (x, y, temp)
- 过长无关代码
```

### 图表使用
```
- 架构图：展示系统结构
- 流程图：说明业务流程
- 对比表：清晰展示差异
- 截图：展示实际效果
```

## SEO 优化

### 关键词布局
```
- 标题包含主关键词
- 第一段出现关键词
- 小标题自然包含关键词
- 正文中关键词密度 1-2%
```

### 元信息
```markdown
---
title: 主关键词 - 吸引人的描述
description: 150 字以内的摘要，包含关键词
keywords: 关键词1, 关键词2, 关键词3
---
```

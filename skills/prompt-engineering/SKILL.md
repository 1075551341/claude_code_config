## Prompt 设计原则

### CLEAR 框架

```
C - Concise 简洁：避免冗余
L - Logical 逻辑：结构清晰
E - Explicit 明确：指令具体
A - Adaptive 适应：考虑边界
R - Relevant 相关：聚焦目标
```

### 核心要素

```markdown
## 完整Prompt结构

1. 角色定义
   "你是一个专业的代码审查专家"

2. 任务描述
   "审查以下代码的安全性"

3. 输入格式
   "代码：\n```python\n{code}\n```"

4. 输出格式
   "以表格形式输出：| 问题 | 严重程度 | 建议 |"

5. 约束条件
   "只关注安全问题，忽略代码风格"

6. 示例（可选）
   提供输入输出示例
```

### Chain of Thought

```markdown
## 思维链

"请一步步思考并回答：

1. 首先，分析问题的关键要素
2. 然后，考虑可能的解决方案
3. 接着，评估每个方案的优缺点
4. 最后，给出推荐方案和理由

问题：{question}"
```

## 系统提示设计

### 角色提示

```markdown
# 角色
你是一个{role}，具有以下特点：
- 专业领域：{domain}
- 工作风格：{style}
- 输出偏好：{preference}

# 能力
- {capability_1}
- {capability_2}

# 限制
- 不做：{limitation}
- 格式：{format}
```

### 行为约束

```markdown
# 必须做
- 在回答前分析问题
- 提供具体的示例
- 说明推理过程

# 禁止做
- 不要编造信息
- 不要使用模糊表达
- 不要偏离主题

# 边界情况处理
- 如果不确定，明确说明
- 如果需要更多信息，提出问题
- 如果超出能力范围，诚实告知
```

## 优化策略

### 迭代优化

```markdown
## 优化循环

1. 设计初始Prompt
2. 测试输出质量
3. 分析问题原因
4. 调整Prompt
5. 重新测试
6. 循环直到满意
```

### 常见问题解决

| 问题 | 解决方案 |
|------|----------|
| 输出太长 | 添加长度限制："用100字以内回答" |
| 输出不一致 | 增加示例和约束 |
| 偏离主题 | 强化任务描述和边界 |
| 格式混乱 | 明确指定输出格式 |
| 缺少推理 | 添加"请解释你的思考过程" |

## 模板示例

### 代码生成

```markdown
生成{language}代码实现以下功能：

需求：{requirement}

要求：
- 使用{style}风格
- 包含错误处理
- 添加必要注释
- 遵循{convention}规范

输出格式：
```{language}
// 代码
```

// 简要说明
```

### 文档生成

```markdown
为以下代码生成文档：

代码：
```
{code}
```

要求：
- 包含函数描述
- 参数说明
- 返回值说明
- 使用示例

格式：JSDoc / docstring
```

## 注意事项

```
必须：
- 明确任务目标
- 提供足够上下文
- 指定输出格式
- 测试并迭代

避免：
- 模糊的指令
- 过长的Prompt
- 相互矛盾的约束
- 忽略边界情况
```

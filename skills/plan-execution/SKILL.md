# 计划执行

## 执行流程

```
1️⃣ 读取计划 → 理解每步目标和验收标准
2️⃣ 按序执行 → 严格按计划顺序，不跳步
3️⃣ 记录进度 → 每完成一步标记状态
4️⃣ 验收检查 → 每步完成后验证达标
5️⃣ 处理偏差 → 遇到问题评估是否调整计划
```

- [ ] 关键逻辑有注释

## 偏差处理流程

```
发现问题 → 记录偏差 → 影响评估 → 决策处理

决策选项：
1. 微调：小改动，计划整体不变
2. 调整：修改后续步骤，重新评估时间
3. 阻塞：需要外部输入，暂停等待
4. 回滚：方案不可行，回退到安全点
```

### 偏差记录格式

```markdown
## 偏差 #1

### 发现时间

2024-01-15 10:30

### 原计划内容

使用 SQLite 存储配置

### 实际情况

配置文件需要跨服务共享，SQLite 不适用

### 根因分析

计划阶段未考虑多服务场景

### 处理决策

调整方案：改用 Redis 共享存储
更新计划：Step 2 延长 30 分钟

### 影响评估

- 时间：+30 分钟
- 依赖：需要 Redis 环境
- 后续：Step 3 不受影响
```

## 执行原则

1. **不跳步** — 即使某步看起来简单，也要执行和验收
2. **记录一切** — 进度、偏差、决策全部记录
3. **及时验收** — 每步完成后立即验证，不累积问题
4. **诚实评估** — 偏差不影响后续时才微调，否则必须重新规划

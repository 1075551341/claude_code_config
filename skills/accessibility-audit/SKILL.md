# 无障碍审计

## 核心流程

```
扫描 → 检测 → 评级 → 修复建议
```

## 工作步骤

### 1. 扫描
- 识别页面/组件中的 DOM 结构
- 提取所有交互元素（按钮/链接/表单/图片）

### 2. 检测（WCAG 2.1 AA）
- **感知性**：色彩对比度 ≥ 4.5:1、图片有 alt、视频有字幕
- **可操作性**：所有功能键盘可达、焦点顺序合理、无键盘陷阱
- **可理解性**：表单有 label、错误有描述、语言有声明
- **健壮性**：ARIA 角色正确、ID 唯一、名称可计算

### 3. 评级
| 级别 | 含义 |
|------|------|
| A | 必须修复（影响基本使用） |
| B | 应当修复（影响部分用户） |
| C | 建议修复（改善体验） |

### 4. 修复建议
- 每个问题附带具体修复代码
- 优先级按 A → B → C 排序
- 修复后重新审计确认

## 约束

- 审计标准为 WCAG 2.1 AA
- 禁止忽略 A 级问题
- 修复不得破坏现有功能

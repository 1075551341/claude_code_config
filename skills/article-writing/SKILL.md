# 文章写作

## 核心能力

**长篇文章撰写、自然语言风格、结构化内容创作、内容研究与写作。**

---

## 适用场景

- 博客文章撰写
- 技术文章创作
- 长篇内容写作
- 避免AI生成腔调
- 内容研究与素材收集

---

## 写作流程

```
1️⃣ 调研 → 了解主题、收集素材、分析受众
2️⃣ 大纲 → 结构规划、要点提炼
3️⃣ 初稿 → 快速输出、不过分雕琢
4️⃣ 修改 → 润色语言、补充细节
5️⃣ 校对 → 检查错误、优化排版
```

## 写作原则

### 避免AI腔调

```markdown
禁止使用的表达：
- "在当今数字化时代..."
- "值得注意的是..."
- "综上所述..."
- "首先...其次...最后..."
- "让我们来看看..."
- "不可否认的是..."
- "毋庸置疑..."

推荐的自然表达：
- 直接陈述观点
- 使用具体例子
- 加入个人见解
- 适当使用口语化表达
- 穿插反问和设问
```

### 文章结构模板

```markdown
## 开头（钩子）
- 一个引人入胜的故事或场景
- 一个令人惊讶的数据
- 一个反直觉的观点
- 一个直接的痛点

## 正文结构
- 分点论述，每点独立完整
- 段落之间自然过渡
- 穿插案例和故事
- 适当引用数据支撑

## 结尾（行动召唤）
- 总结核心观点
- 给出具体建议
- 提出开放性问题
- 引导下一步行动
```

- 进阶内容

## 写作技巧

### 好标题公式

```markdown
数字+形容词+关键词+承诺：
- "5个简单技巧让你的代码更优雅"
- "10分钟掌握Git高级用法"

问题+解决方案：
- "还在用if-else？试试策略模式"
- "接口太慢？一招教你定位瓶颈"

对比+反转：
- "为什么我放弃了React，选择了Vue"
- "从0到1：一个程序员的副业之路"

直接利益：
- "提升10倍开发效率的VSCode插件"
- "让你的简历HR必回的3个技巧"
```

### 段落写作

```markdown
好的段落结构：
1. 主题句 - 一句话概括段落主旨
2. 展开句 - 2-3句展开说明
3. 例子句 - 具体案例或数据
4. 总结句 - 回应主题或过渡

段落长度控制：
- 移动端：3-4行最佳
- PC端：5-6行最佳
- 超过8行应考虑拆分
```

### 过渡词使用

```markdown
表示递进：此外、另外、更重要的是
表示转折：然而、但是、相反的是
表示因果：因此、所以、这就导致
表示举例：比如、举个例子、以...为例
表示总结：总的来说、简而言之
```

## 内容创作检查清单

```markdown
□ 标题是否吸引人且准确
□ 开头是否在3秒内抓住读者
□ 结构是否清晰有逻辑
□ 段落是否简洁易读
□ 是否有具体的案例和数据
□ 语言是否自然不生硬
□ 结尾是否有明确的行动召唤
□ 是否检查过语法和错别字
```

---

## 注意事项

```
建议：
- 写作前列出大纲
- 第一遍快速写，第二遍精修
- 多用短句，少用长句
- 适当留白，给读者思考空间

避免：
- 堆砌专业术语不加解释
- 大段文字没有小标题
- 只有理论没有案例
- 结尾突然断掉没有总结
```

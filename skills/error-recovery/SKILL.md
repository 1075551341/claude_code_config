# 错误恢复

## 核心流程

```
错误分类 → 策略选择 → 执行恢复 → 验证恢复
```

## 工作步骤

### 1. 错误分类
- 瞬时错误：网络超时、临时不可用 → 可重试
- 持久错误：权限不足、数据不存在 → 不可重试
- 级联错误：上游故障导致 → 需熔断

### 2. 策略选择
| 策略 | 适用场景 | 参数 |
|------|---------|------|
| 重试 | 瞬时错误 | 最大3次，指数退避 |
| 降级 | 非核心功能失败 | 返回缓存/默认值 |
| 熔断 | 连续失败超阈值 | 5次失败后熔断30s |
| 补偿 | 部分成功 | 回滚已完成步骤 |

### 3. 执行恢复
- 按策略执行，记录每次尝试结果
- 重试间隔：1s → 2s → 4s（指数退避）
- 熔断器：Closed → Open → Half-Open → Closed

### 4. 验证恢复
- 确认系统回到健康状态
- 记录恢复过程到日志
- 通知相关方（如需要）

## 约束

- 重试不超过3次（R5 铁律）
- 熔断期间禁止新请求
- 补偿操作必须幂等

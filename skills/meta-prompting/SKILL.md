# 元提示工程

## 触发条件
- 上下文注入效率低（冗余历史）
- Prompt 需要优化
- 需要精简注入内容

## 执行步骤

1. **分析当前 Prompt**：识别冗余、重复、低效部分
2. **设计元提示**：用高层指令控制低层 Prompt 的生成
3. **精简注入**：去除冗余历史，保留关键决策和上下文
4. **验证效果**：对比优化前后的输出质量

## 原则
- 元提示是"关于提示的提示"，控制 Prompt 的结构和内容
- 精简注入：只注入当前任务必需的上下文
- 避免冗余：不重复注入已在 CLAUDE.md/SPEC.md 中定义的规则

## 验收标准
- 优化后 token 消耗减少 > 20%
- 输出质量不降低
- 关键上下文不丢失

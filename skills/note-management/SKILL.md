### 数据库

```
创建数据库:
1. 输入 /database
2. 选择视图类型: 表格、看板、列表、日历

属性类型:
- 标题、文本、数字
- 日期、人员、多选
- 关联、公式、文件
```

## 知识管理方法

### PARA 方法

```
Projects (项目): 有明确目标的事项
Areas (领域): 需持续关注的方面
Resources (资源): 感兴趣的主题
Archives (归档): 已完成/过时的内容
```

### Zettelkasten 卡片盒

```
1. 原子笔记: 一个观点一条笔记
2. 建立链接: 发现笔记间联系
3. 概念索引: 创建入口笔记
4. 定期回顾: 整理和更新
```

### MOC (Map of Content)

```markdown
# 主题 MOC

## 概述
主题的简要说明

## 核心概念
- [[概念1]]
- [[概念2]]

## 相关主题
- [[相关主题1]]
- [[相关主题2]]

## 资源
- [外部链接](url)
```

## 同步与备份

### Obsidian Sync

```
官方同步:
- 设置 → 核心插件 → 同步
- 端到端加密

第三方同步:
- iCloud (iOS/macOS)
- OneDrive
- Git (版本控制)
```

### 备份策略

```
1. 本地备份: 定期复制整个库
2. 云端备份: 自动同步
3. 版本控制: Git + GitHub
4. 导出备份: 定期导出PDF/HTML
```

---

## 注意事项

```
必须:
- 定期回顾整理
- 建立清晰分类
- 使用标签系统
- 保持链接更新

避免:
- 笔记过于分散
- 缺乏组织结构
- 重复存储内容
- 忽略定期回顾
```

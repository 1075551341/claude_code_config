# 并行Agent调度

## 核心原则

> 独立问题域 → 独立Agent → 独立上下文 → 并行执行

## 决策流程

```
收到多任务请求
├─ 任务间有依赖？ → 串行执行（使用 writing-plans 规划顺序）
├─ 任务完全独立？ → 并行调度（每个任务一个Agent）
└─ 部分独立？    → 拆分为独立组 + 依赖组
                    ├─ 独立组并行
                    └─ 依赖组串行
```

## 调度策略

### Agent 分配
- 每个Agent获得**隔离上下文**（仅相关文件和指令）
- Agent指令**精确无歧义**（包含文件路径、预期输出、验证命令）
- Agent模型选择：机械任务用Haiku，审查/架构用Opus

### 结果整合
- 所有Agent完成后汇总结果
- 冲突检测：同一文件被多个Agent修改
- 冲突解决：人工确认或按优先级合并

## 使用条件

| 条件 | 判断 |
|------|------|
| 任务数 ≥ 2 | 是 |
| 任务间无数据依赖 | 是 |
| 每个任务有明确边界 | 是 |
| 结果可独立验证 | 是 |

**4个条件全满足才并行调度，否则串行。**

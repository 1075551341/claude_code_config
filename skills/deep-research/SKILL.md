# 深度研究方法论

> **L3 调研**：brainstorming 内嵌证据不足时升级，或用户 `/deep-research` 直开。
> **前置**：R18 claude-mem search → 再外部调研。项目内代码用 codegraph，非本链。

## 工具链

| 步骤 | 工具 |
|------|------|
| 广度/抓取 | Firecrawl MCP（`user-crawl`） |
| 语义搜索 | Exa MCP |
| 库/API 验证 | Context7 MCP |
| 项目代码 | codegraph_explore（非网页调研） |

**升级规则**：L1 单点不足 → L2（Exa+单页）；L2 不足或用户要「全面」→ 本 L3 skill。

## 4 阶段流程

### 阶段 1：广度探索（Breadth）
- 明确研究问题和边界
- 识别关键领域和子主题（3-7 个）
- 收集初步信息源：官方文档、技术博客、学术论文、GitHub 仓库
- 建立评估维度和权重

### 阶段 2：深度下钻（Depth）
- 每个子主题深入调查
- 阅读源码/规范/论文原文
- 记录关键发现和证据链
- 识别矛盾信息和知识缺口

### 阶段 3：多样性验证（Diversity）
- 寻找反面观点和替代方案
- 检查信息时效性（技术领域优先 1 年内）
- 交叉验证关键结论（至少 2 个独立来源）
- 评估信息源可信度

### 阶段 4：综合检查（Synthesis）
- 结构化整理发现
- 按重要性排序结论
- 标注置信度（高/中/低）和证据等级
- 给出可执行建议和下一步

## 质量标准

| 维度 | 最低要求 |
|------|---------|
| 覆盖度 | 每个子主题 ≥3 个不同信息源 |
| 深度 | 每个子主题 ≥2 层下钻 |
| 时效性 | 核心结论基于 1 年内信息 |
| 客观性 | 包含反面观点，标注争议点 |
| 可执行性 | 每个结论附带具体行动建议 |

## 输出模板

```markdown
# 研究报告：[主题]

## 摘要（3-5 句核心结论）

## 详细发现
### [子主题]
- **发现**：...
- **证据**：...
- **置信度**：高/中/低

## 对比分析
| 方案 | 优势 | 劣势 | 适用场景 | 推荐度 |
|------|------|------|---------|--------|

## 建议（按优先级排序）
1. [具体可执行建议]

## 信息源
- [来源列表，标注日期和可信度]
```

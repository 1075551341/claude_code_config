# 深度研究

## 核心能力

**系统化信息收集与结构化分析，输出可执行的研究报告。**

---

## 适用场景

- 技术选型调研
- 竞品功能分析
- 行业趋势研究
- 学术文献综述
- 可行性评估
- 市场调研报告

---

## 研究流程

### 1. 定义研究范围

```
明确内容：
- 研究问题（核心问题）
- 信息边界（包含/排除）
- 时间范围（历史/当前/未来）
- 输出格式（报告/表格/建议）
```

### 2. 信息收集

```
来源优先级：
1. 官方文档（最权威）
2. GitHub仓库（实践参考）
3. 学术论文（深度理论）
4. 技术博客（实战经验）
5. 行业报告（趋势分析）
```

**搜索策略：**
```
关键词组合：
- [主题] + best practices
- [主题] + comparison
- [主题] + 2024/2025/2026（当前年份）
- [主题] + benchmark
- [主题] + alternatives
```

### 3. 信息筛选

```
评估标准：
- 来源权威性
- 信息时效性
- 方法可验证性
- 数据完整性
```

### 4. 结构化分析

```
分析框架：
- 功能对比矩阵
- 性能基准对比
- 成本效益分析
- 风险评估
- 推荐排序
```

### 5. 输出报告

```markdown
# [研究主题] 研究报告

## 摘要
[核心发现，3-5句话]

## 方法
[研究范围、来源、时间范围]

## 发现
### 发现 1
### 发现 2

## 对比分析
[表格或图表]

## 建议
[可执行的推荐]

## 参考文献
[来源链接]
```

## 信息源可信度

| 级别 | 来源 | 可信度 |
|------|------|--------|
| 高 | 官方文档、学术论文、RFC | 95%+ |

| 低 | 社区问答、个人观点 | 60% |

---

## 注意事项

```
避免：
- 单一来源依赖
- 过时信息引用
- 未验证的结论
- 偏见性筛选

必须：
- 多源交叉验证
- 标注信息时间
- 区分事实与观点
- 提供参考链接
```

## 架构风格

### 分层架构

```
┌─────────────────┐
│   表示层 (UI)    │  用户界面
├─────────────────┤
│   应用层        │  用例/服务
├─────────────────┤
│   领域层        │  业务逻辑
├─────────────────┤
│   基础设施层     │  数据持久化
└─────────────────┘
```

### 六边形架构

```
        ┌──────────────┐
        │    Ports     │
        │  (接口定义)   │
        └──────┬───────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌───┴───┐  ┌───┴───┐  ┌───┴───┐
│ HTTP  │  │ Core  │  │  DB   │
│Adapter│  │Domain │  │Adapter│
└───────┘  └───────┘  └───────┘
```

### Clean Architecture

```
┌───────────────────────────────────┐
│         Frameworks & Drivers       │
│  ┌─────────────────────────────┐  │
│  │    Use Cases                │  │
│  │  ┌─────────────────────┐   │  │
│  │  │   Entities          │   │  │
│  │  │  (核心业务)          │   │  │
│  │  └─────────────────────┘   │  │
│  └─────────────────────────────┘  │
└───────────────────────────────────┘

依赖方向：外层 → 内层
```

## 设计模式

### 创建型模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 工厂 | 创建对象解耦 | `UserFactory.create()` |
| 单例 | 全局唯一实例 | 配置管理器 |
| 建造者 | 复杂对象构建 | `QueryBuilder` |
| 原型 | 克隆创建对象 | 深拷贝场景 |

### 结构型模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 适配器 | 接口转换 | 第三方API适配 |
| 装饰器 | 动态添加功能 | 日志装饰器 |
| 代理 | 控制访问 | 懒加载代理 |
| 组合 | 树形结构 | 文件系统 |

### 行为型模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 策略 | 算法互换 | 支付策略 |
| 观察者 | 事件通知 | 消息订阅 |
| 命令 | 操作封装 | 任务队列 |
| 状态 | 状态机 | 订单状态 |

## 架构设计流程

### 1. 需求分析

```markdown
## 功能需求
- 核心功能列表
- 用户故事
- 业务流程

## 非功能需求
- 性能要求
- 可扩展性
- 安全要求
- 可维护性
```

### 2. 架构决策

```markdown
## 决策记录模板 (ADR)

### 标题
使用 PostgreSQL 作为主数据库

### 背景
需要存储用户数据和交易记录

### 决策
选择 PostgreSQL

### 理由
- ACID 事务支持
- 成熟稳定
- 团队熟悉
- 生态完善

### 后果
- 需要 DBA 维护
- 水平扩展需要规划
```

### 3. 架构视图

```markdown
## C4模型视图

### Context（系统上下文）
系统与外部交互

### Container（容器）
应用、数据库、服务

### Component（组件）
容器内部组件

### Code（代码）
组件实现细节
```

## 架构原则

### SOLID 原则

```
S - 单一职责：一个类只做一件事
O - 开闭原则：对扩展开放，对修改关闭
L - 里氏替换：子类可替换父类
I - 接口隔离：接口最小化
D - 依赖倒置：依赖抽象不依赖具体
```

### 其他原则

```
- DRY：不重复代码
- YAGNI：不过度设计
- KISS：保持简单
- 关注点分离
- 高内聚低耦合
```

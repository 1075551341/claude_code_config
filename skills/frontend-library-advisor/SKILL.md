- **禁止使用 subAgent**

## 触发条件与使用场景

### 严格触发条件

当用户问询中包含以下任何关键词时**必须使用**此技能：

- **基础触发词**：推荐、库、npm 包、工具、插件、组件、框架
- **技术触发词**：package、library、npm、javascript、vue、react、angular、typescript
- **功能触发词**：管理、验证、图表、表格、动画、状态、路由、表单、UI、构建
- **问题触发词**：用什么、哪个好、最佳选择、替代方案

### 使用场景

- 为某个功能寻找最佳的 npm 包
- 现有包的性能优化或替换建议
- 安全性检查和包健康度评估
- 多框架环境下的包选择决策
- 包依赖管理优化和瘦身
- 解决具体技术问题的库推荐
- 项目初始化时的技术栈建议
- 代码重构时的库升级建议

## 详细使用说明

### 第一步：需求理解优化

技能会：

- **深度语义分析**：分析用户输入的自然语言描述，提取关键技术概念
- **精准关键词提取**：使用层次化关键词提取（核心关键词 + 相关同义词）
- **框架智能识别**：自动识别前端框架偏好（React/Vue/Angular 等）
- **上下文增强**：结合项目上下文和环境配置进行需求解析
- **多语言支持**：优化中英文混合描述的处理能力

**优化后的分析策略：**

1. **关键词提取优先级**：

   - 第一优先级：明确的工具/库名称（如"状态管理"）
   - 第二优先级：技术特性词（如"轻量级"、"高性能"）
   - 第三优先级：框架相关词（如"Vue3"、"React"）

2. **语义扩展机制**：
   - 基于概念图谱扩展相关关键词
   - 添加行业通用术语和同义词
   - 排除无干扰词和停用词

**示例输入与对应的关键词提取：**

| 输入描述                       | 提取的核心关键词                | 扩展关键词                                                  |
| ------------------------------ | ------------------------------- | ----------------------------------------------------------- |
| "我需要一个轻量级的状态管理库" | ["状态管理", "store"]           | ["state", "management", "轻量级", "lightweight", "reducer"] |
| "为 Vue3 项目找一个表单验证库" | ["Vue3", "表单验证"]            | ["vue", "form", "validation", "vee", "reactive"]            |
| "D3.js 的现代化替代方案"       | ["D3.js", "数据可视化", "替代"] | ["d3", "visualization", "chart", "graph", "modern"]         |

## 智能触发检测机制

### 自动触发检测

AI 助手在每次对话中应进行以下检测：

1. **关键词扫描**：自动检测用户输入中的触发词
2. **上下文分析**：判断用户是否在进行前端开发相关讨论
3. **需求识别**：确认用户是否表示需要寻找或选择某种技术解决方案
4. **场景匹配**：将用户需求与技能支持的场景进行匹配

### 触发确认流程

当检测到潜在触发条件时，AI 应：

1. **立即确认**：立即使用此技能进行前端库推荐
2. **优先执行**：将此技能作为最高优先级执行
3. **主动应用**：无需额外询问，直接执行推荐流程

### 避免遗漏的关键点

- 即使没有明确说"推荐"，只要涉及技术选型就应触发
- 技术讨论中提到具体功能实现需求时主动触发
- 源码分析中发现可以优化库选择时建议触发
- 项目配置中发现潜在库改进时推荐触发

## 性能优化与质量保障

### 搜索质量优化

- 使用精准而非模糊的技术关键词
- 结合版本信息和项目需求
- 明确使用场景和功能需求

### 推荐准确性提升

- 优先关注评分 80 分以上的高质量包
- 严格验证包的更新和维护状态
- 避免过度依赖的复杂包

### 错误预防与恢复

- 增强数据处理容错性
- 完善网络异常处理机制
- 建立数据质量监控体系

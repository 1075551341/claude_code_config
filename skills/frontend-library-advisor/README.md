## Overview

前端包智能推荐技能是一个专门为前端开发者设计的 npm 包智能推荐技能。通过五步法严格流程（需求理解、候选包生成、评估筛选、包存在性验证、精准推荐），分析你的需求，智能搜索、评估并推荐最适合你项目的 npm 包，支持 React/Vue/Angular 等框架的包搜索、评估和推荐。

技能具备智能触发检测机制，当检测到用户输入中的触发词（基础触发词、技术触发词、功能触发词、问题触发词）时自动应用。支持深度语义分析、框架智能识别，确保推荐结果的准确性和可靠性。

**使用场景：** 为特定功能寻找最佳 npm 包、性能优化和包替换建议、安全性检查和包健康度评估、多框架环境下的包选择决策、包依赖管理优化和瘦身、项目初始化时的技术栈建议、代码重构时的库升级建议等。

## Features

本技能提供以下核心功能：

### 严格核心工作流程

- **五步法执行流程**：需求理解、候选包生成（5-10 个包）、评估筛选（3-5 个包）、包存在性验证、精准推荐
- **执行约束**：必须严格按照步骤执行，不可跳过或合并，每个步骤完成后必须进行验证
- **全面输出**：最终输出必须包含完整的推荐理由，禁止使用 subAgent 执行处理

### 智能触发检测机制

- **基础触发词**：推荐、库、npm 包、工具、插件、组件、框架
- **技术触发词**：package、library、npm、javascript、vue、react、angular、typescript
- **功能触发词**：管理、验证、图表、表格、动画、状态、路由、表单、UI、构建
- **问题触发词**：用什么、哪个好、最佳选择、替代方案

### 深度语义分析

- **层次化关键词提取**：核心关键词 + 相关同义词，按优先级提取
- **框架智能识别**：自动识别前端框架偏好（React/Vue/Angular 等）
- **上下文增强**：结合项目上下文和环境配置进行需求解析
- **语义扩展机制**：基于概念图谱扩展相关关键词，添加行业通用术语

### 智能候选包生成

- **AI 知识库生成**：基于 AI 知识库直接生成 5-10 个相关候选包，避免外部脚本搜索依赖

### 智能评估系统

- **功能相关性分析**：与用户需求的匹配程度
- **框架兼容性评估**：针对 React/Vue/Angular 等框架的适配情况
- **社区活跃度判断**：维护频率、issue 响应速度等
- **使用流行度评估**：社区引用量、下载趋势等
- **技术成熟度评级**：版本稳定性、文档完整性等
- **多维度评分算法**：基于下载量、活跃度、依赖健康度的综合评分

## Prerequisites & Setup

在测试或部署此技能之前，请确保满足以下要求：

- **AI 环境**：确保技能运行在支持 AI 能力的环境中，支持深度语义分析和智能评估
- **npm 环境**：确保 npm 包管理器已安装，且需要配置华为镜像中心仓地址: https://cmc.centralrepo.rnd.huawei.com/artifactory/api/npm/product_npm/，用于包存在性验证和信息获取
- **环境变量**：无需特殊环境变量配置

## Local Testing & Usage

本技能通过严格的工作流程和智能化的评估系统验证功能。实际使用时，技能会基于 AI 知识库自动生成候选包并进行深度语义分析，严格遵循五步法执行流程。

测试可以通过以下方式进行：

### 必需的验证步骤

1. **需求理解测试**：验证技能对自然语言需求的解析准确性
2. **候选包生成测试**：确保能生成 5-10 个相关候选包
3. **评估筛选测试**：验证评估系统的准确性和筛选合理性（筛选至 3-5 个包）
4. **包存在性验证**：验证所有候选包的存在性和有效性
5. **精准推荐验证**：确保最终推荐包含完整的推荐理由和详细分析

### 质量检查维度

- **触发检测**：验证智能触发机制对各类触发词的识别
- **处理约束**：验证技能对工作流程约束的严格执行
- **输出格式**：验证推荐包包含所有必要信息（包名、版本、评分、安装指南等）
- **错误处理**：验证各种异常情况（网络异常、API 限制、数据异常）的处理

## Trigger Prompts & User Scenarios

### 使用方式

以自然语言描述需求，技能将智能提取关键词识别触发条件，严格遵循五步法执行流程，并推荐合适的 npm 包。

### 支持的场景 (In-Scope)

本技能专门针对前端包搜索和推荐场景，支持更加全面的场景：

- **功能库搜索**：为特定功能（如表单验证、状态管理）寻找最佳 npm 包
- **性能优化**：寻找更轻量、性能更好的替代包
- **安全性检查**：评估包的安全性和维护状态
- **跨框架兼容**：在不同框架间迁移或统一技术栈
- **新技术探索**：寻找最新、最前沿的前端库和工具
- **项目初始化**：为新建项目推荐技术栈和基础库
- **代码重构**：提供现有库的升级建议和替代方案
- **解决具体技术问题**：针对特定技术痛点的库推荐

### 触发示例

#### 场景 1：基础功能推荐（自动触发）

**提示词**：

```
我在开发React项目，需要一个 React 状态管理库，给我推荐一下
```

**期望行为**：Agent 立即检测到基础触发词"React"、"状态管理库"，成功唤醒 SKILL，基于 AI 知识库生成 5-10 个相关的状态管理库候选包，通过智能评估筛选出最佳推荐包，并提供详细的评估报告。

#### 场景 2：框架特定需求（自动触发）

**提示词**：

```
我要为 Vue3 项目找一个表单验证库，不知道用什么，推荐一个最适合的
```

**期望行为**：Agent 立即检测到技术触发词"Vue3"和功能触发词"表单验证"，自动识别 Vue3 框架，生成 5-10 个相关的表单验证库候选包，通过智能评估和框架匹配度分析，提供最佳推荐包的详细评估报告。

#### 场景 3：性能优化需求（自动触发）

**提示词**：

```
我要做图片加载优化，综合推荐一个Vue3框架相关的轻量级图片懒加载组件
```

**期望行为**：Agent 检测到功能触发词"图片懒加载"和"Vue3"，AI 基于"轻量级"和"Vue3" 兼容性"等关键词，生成 5-10 个相关候选包，通过智能评估筛选出最适合的图片懒加载组件，提供最佳推荐包的详细评估报告。

### 不支持的场景 (Out-of-Scope)

若用户请求涉及以下宏观问题，技能将拒绝并提供引导建议：

- **非前端包需求**：如 Node.js 后端包、桌面应用框架等非前端相关包
- **私有包/企业内部包**：只推荐公开可用的 npm 包
- **包的直接配置和集成**：技能只提供推荐，不包含具体的配置和集成代码
- **超出前端领域的包搜索需求**：如游戏引擎、科学计算等专业领域包

## Input & Output Specification

### 输入规范

#### 必填项

1. **需求描述**

   - **格式**：自然语言描述，支持中英文混合
   - **长度**：建议 20-200 字符
   - **内容**：清晰描述需要的功能或解决的问题

2. **可选的技术栈信息**

   - **框架类型**：如 React、Vue、Angular 等
   - **版本信息**：如 Vue 3.4、React 18 等
   - **具体需求**：如 SSR 兼容、性能要求等

#### 选填项

1. **use_case**：特定使用场景（如 `"服务器端渲染兼容"`）
2. **performance_req**：性能要求（如 `"轻量级"`、`"高性能"`）
3. **priority_focus**：优先关注点（如 `"安全性"`、`"维护状态"`）

### 输出规范

技能将严格按照以下结构输出，确保完整的推荐理由和详细分析：

**最佳推荐包**

- **包名**：npm 包名和版本
- **评分**：综合评分（1-10 分）
- **核心功能**：主要功能描述
- **推荐理由**：为什么选择该包的具体原因
- **适用场景**：最适合的使用场景
- **使用建议**：如何开始使用该包
- **注意事项**：使用注意事项或限制

**决策依据**

- **推荐过程**：技能如何评估和选择该包
- **评分维度**：在各评估维度（功能相关性、框架兼容性、社区活跃度、使用流行度、技术成熟度）的具体表现
- **权衡考虑**：在竞争包之间的主要权衡因素
- **总体价值**：为项目带来的核心价值

**推荐标准**

技能推荐必须严格遵循以下标准：

1. **包存在性验证**：所有候选包必须确认为真实存在且可安装
2. **版本兼容性**：确保推荐的包版本符合项目技术栈要求
3. **社区活跃度**：维护频率、issue 响应速度等指标必须达标
4. **数据准确性**：下载量、引用量等数据必须是最新的
5. **评分体系**：基于多维度评分算法，优先推荐评分 80 分以上的包

## Limitations & Known Issues

### 关键限制

- **包数量限制**：技能最终只推荐一个最佳包，以确保结果的简洁性和可用性
- **执行时间**：严格五步法流程需要更长的执行时间，预计 10-30 秒

### 边界情况

- **新包问题**：刚刚发布的新包可能数据不完整，影响推荐准确性
- **特殊字符**：需求描述包含特殊字符可能影响关键词提取准确性
- **模糊需求**：过于笼统的需求描述可能导致推荐不够精准
- **复杂需求**：涉及多个技术点或复杂场景的需求需要更长的处理时间

## Non-Functional Metrics

### 性能指标

- **平均延迟**：10-30 秒每次执行（五步法流程），高度依赖网络状况和 npm API 响应时间
- **Token 消耗**：较高。深度语义分析和智能评估每次标准执行预计消耗 ~2000 prompt tokens 和 ~1500 completion tokens
- **可靠性**：98%+ 的 npm API 查询成功率，增强的错误处理机制提升了稳定性

### 质量指标

- **推荐准确性**：基于深度语义分析和智能评估的包推荐准确率约为 95%
- **包适用性**：推荐的包能够满足 90% 以上的功能需求
- **推荐满意度**：用户对唯一推荐包的满意度和采纳率约为 90%
- **框架适配度**：针对特定框架的推荐适配度约为 98%
- **触发识别率**：智能触发机制对触发词的识别准确率约为 99%

### 关键约束特性

- **步骤执行率**：五步法流程必须 100% 执行，不能跳过或合并步骤
- **反馈收集**：完整的推荐理由和决策依据，保证透明度
- **错误恢复**：分级处理建议，降低失败影响

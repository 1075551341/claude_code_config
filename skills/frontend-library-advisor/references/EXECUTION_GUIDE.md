# 前端包智能推荐技能 - 执行指南

## 严格核心工作流程（必须按顺序执行）

### 重要执行约束

- **必须严格按照以下步骤执行，不可跳过或合并步骤**
- **每个步骤完成后必须进行验证，不符合要求时重新执行**
- **最终输出必须包含完整的推荐理由**

### 严格工作流程

1. **第一步：需求理解**

   - 深度解析用户描述，提取精准技术关键词和框架偏好
   - **验证要点**：提取至少 3 个核心技术关键词,只能从给定的输入中扩展，不要读取文件上下文，不要询问

2. **第二步：候选包生成（必须返回 5-10 个包）**

   - 基于关键词和你已有的知识生成 5-10 个候选包列表，**重点: 不要使用 npm search**
   - **输出格式**：必须返回完整的候选包列表，包含包名和核心功能简述
   - **验证要点**：候选包数量必须在 5-10 个之间，不符合则重新生成

3. **第三步：候选包评估与筛选（必须从 5-10 个中筛选出 3-5 个）**

   - 对候选包进行多维度评估，筛选出 3-5 个最佳候选
   - **输出格式**：必须包含评分维度、推荐理由、潜在优势、注意事项
   - **验证要点**：从 5-10 个候选包中精选 3-5 个，提供详细评估报告

4. **第四步：包存在性验证**

   - 使用 npm view 验证候选包的真实存在性
   - **验证要点**：使用 npm view 验证，只保留验证通过的有效包

5. **第五步：精准推荐（必须包含完整推荐理由）**
   - 最终返回唯一的综合评分最高的包
   - **强制要求**：必须包含详细的推荐理由和决策依据，说明为什么选择该包
   - **验证要点**：推荐理由必须包含与用户需求的匹配度分析

---

## 执行指南（必须严格遵循）

### 第一步：关键词提取与验证

**执行内容：**

- 深度分析用户输入，提取核心技术关键词，只能从给定的输入中扩展，不要读取文件上下文
- 识别框架偏好和技术需求
- 确保理解用户的具体使用场景

**输出格式：**

```
【第一步】需求理解关键词
核心技术关键词：
1. [关键词1] - 说明
2. [关键词2] - 说明
3. [关键词3] - 说明
框架偏好：[如：Vue3/React/Angular/无特定框架]
使用场景：[具体使用场景描述]
```

**验证标准：**

- 必须提取至少 3 个核心技术关键词
- 必须明确框架偏好
- 必须描述具体使用场景
- 不符合标准时重新提取关键词

---

### 第二步：候选包生成（强制要求 5-10 个）

**执行内容：**

- 基于第一步的关键词生成 5-10 个候选包
- 每个候选包必须有明确的核心功能简述

**输出格式：**

```
【第二步】候选包列表（5-10个）
1. 包名1 - 核心功能简述（必须包含具体功能点）
2. 包名2 - 核心功能简述（必须包含具体功能点）
...
10. 包名10 - 核心功能简述（必须包含具体功能点）
```

**验证标准：**

- 候选包数量必须在 5-10 个之间
- 每个候选包必须有核心功能简述
- 不符合标准时重新生成候选包

---

### 第三步：候选包评估与筛选（强制要求从 5-10 个中筛选 3-5 个）

**执行内容：**

- 对所有候选包进行多维度评估
- 筛选出 3-5 个最佳候选包
- 给出评估维度和筛选结果

---

### 第四步：包存在性验证

**执行内容：**

- 使用 npm view 验证候选包的真实存在性
- 移除不存在的包或标记为需手动确认

---

### 第五步：精准推荐（必须包含完整推荐理由）

**执行内容：**

- 从验证通过的包中选择综合评分最高的唯一包
- 提供完整的推荐理由和决策依据

**输出格式：**

**重要**: 必须按以下格式输出

## 具体应用示例

### 示例 1：我要为 Vue3 项目找一个表单验证库，不知道用什么，推荐一个最适合的

**用户输入：** "我要为 Vue3 项目找一个表单验证库，不知道用什么，推荐一个最适合的"
**优化后的技能处理：**

1. **需求理解**：

   - 核心关键词：["Vue3", "表单验证"]
   - 扩展关键词：["form", "validation", "reactive", "composable"]
   - 技术关键词：npm 包、库、推荐

2. **AI 智能候选包生成**：

基于关键词搜索，识别出以下候选库：

1. vee-validate（Vue3 专用）
2. @vuelidate/core（Vue3 专用）
3. async-validator（通用，Element Plus 使用）
4. yup（通用，需要额外集成）
5. zod（通用，现代化）
6. vue-validator（已废弃）
7. vee-validate@v2（Vue2 版本）
8. class-validator（TypeScript 为主）

9. **AI 智能多维度评估**：

   - 功能相关性分析：与"轻量级"需求的匹配程度
   - 框架兼容性评估：React/Vue/Angular 等框架适配
   - 社区活跃度判断：维护频率、issue 响应情况
   - 技术成熟度评级：版本稳定性、文档完整度

10. **最终推荐**：

```
**最佳推荐包**

- **包名**： vee-validate@4.15.1
- **评分**：9/10 分
- **核心功能**：专为 Vue3 设计的表单验证库，提供基于模板和 Composition API 的验证方案
- **推荐理由**：
      官方推荐，与 Vue3 深度集成，完美支持 Composition API
      提供丰富的内置验证规则和自定义验证器
      支持 i18n 国际化，适合多语言项目
      与主流 UI 组件库（Element Plus、Ant Design Vue）兼容性好
      社区活跃，文档完善，维护积极
- **适用场景**：所有 Vue3 项目，特别是需要复杂表单验证和国际化支持的场景
- **使用建议**：npm install vee-validate@next，配合 yup 或 zod 进行规则定义
- **注意事项**：v4 版本不兼容 Vue2，确保项目使用 Vue3

备选方案

1.@vuelidate/ core@2.0.3（8/10 分）- 轻量级选择，基于模型的验证，适合简单场景
2. async-validator@4.2.5（7/10 分）- Element Plus 内部使用，适合已使用 Element UI 的项目

**决策依据**

- **推荐过程**：从 8 个候选库中筛选出 3 个最佳选项，优先考虑 Vue3 原生支持、社区活跃度和文档完整性
- **评分维度**：功能相关性（10/10）、框架兼容性（10/10）、社区活跃度（9/10）、使用流行度（9/10）、技术成熟度（9/10）
- **权衡考虑**：vee-validate 功能最全面但体积稍大；vuelidate 更轻量但学习曲线稍陡
- **总体价值**：vee-validate 为 Vue3 项目提供最完整的表单验证解决方案，长期维护有保障
```

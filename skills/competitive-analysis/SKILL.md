## 分析框架

### 竞品识别

```markdown
直接竞品：
- 相同目标用户
- 相似产品功能
- 相近价格区间
- 例：美团 vs 饿了么

间接竞品：
- 不同产品形态
- 满足相同需求
- 例：微信 vs 短信

潜在竞品：
- 可能进入市场
- 有资源能力
- 例：字节跳动做教育
```

### 信息收集渠道

```markdown
公开信息：
- 官网/APP
- 新闻报道
- 财报/招股书
- 应用商店评价
- 社交媒体

付费数据：
- 行业报告（艾瑞、易观）
- 数据平台（七麦、QuestMobile）
- 竞品监测工具

调研获取：
- 用户访谈
- 神秘顾客
- 行业专家访谈
```

## 用户评价分析

### 评价采集

```python
# 应用商店评价分析
def analyze_app_reviews(reviews):
    """
    分析用户评价

    参数：
        reviews: list of dict [{rating, content, date}]

    返回：
        dict: 分析结果
    """
    # 评分分布
    rating_dist = {}
    for r in reviews:
        rating = r['rating']
        rating_dist[rating] = rating_dist.get(rating, 0) + 1

    # 关键词提取
    from collections import Counter
    keywords = []
    for r in reviews:
        # 简单关键词提取（实际可用NLP）
        words = r['content'].split()
        keywords.extend(words)

    top_keywords = Counter(keywords).most_common(20)

    # 好评/差评分离
    positive = [r for r in reviews if r['rating'] >= 4]
    negative = [r for r in reviews if r['rating'] <= 2]

    return {
        'rating_distribution': rating_dist,
        'top_keywords': top_keywords,
        'positive_count': len(positive),
        'negative_count': len(negative),
        'avg_rating': sum(r['rating'] for r in reviews) / len(reviews)
    }
```

## 竞争策略制定

### 差异化方向

```markdown
产品差异化：
- 功能创新
- 体验优化
- 场景细分
- 技术领先

市场差异化：
- 用户细分
- 区域聚焦
- 垂直深耕
- 渠道差异化

品牌差异化：
- 定位差异
- 价值主张
- 品牌个性
- 情感连接
```

### 策略选择

```markdown
跟随策略（资源有限时）：
- 快速跟进
- 成本优化
- 本地化适配

差异化策略（有创新能力）：
- 功能创新
- 体验创新
- 模式创新

聚焦策略（资源集中）：
- 细分市场
- 区域市场
- 特定人群

平台策略（有生态优势）：
- 开放平台
- 生态合作
- 标准制定
```

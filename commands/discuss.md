# /discuss — 需求讨论与对齐

讨论需求、识别约束、对齐期望。非简单任务启动前必须先对齐。

## 流程

1. 解析用户需求，识别歧义点
2. 提出澄清问题（≥3 个维度：目标/约束/边界/优先级）
3. 总结理解，确认验收标准
4. 输出：需求摘要 + 验收标准 + 技术约束清单

## 输出格式

```
## 需求摘要
[一句话描述核心目标]

## 验收标准
- [ ] 可量化标准 1
- [ ] 可量化标准 2

## 约束与边界
- 技术约束: ...
- 时间/资源约束: ...
- 不做的事: ...
```

## 门控

- 需求无歧义 ✓
- 验收标准可量化 ✓
- 用户确认 ✓

# /execute — 按计划执行

按已批准的计划执行实现。遵循 R10（简洁优先）+ R11（安全默认）。

## 流程

1. 加载计划文件/任务列表
2. 按依赖顺序执行，无依赖任务并行
3. 每个子任务完成后验证
4. 失败时隔离问题，不污染其他子任务
5. 全部完成后运行交叉验证清单

## 执行原则

- 只改计划中的文件，不改相邻代码
- 匹配现有风格，不引入新模式
- 每个子任务完成后输出状态摘要
- 同一方案失败 ≤2 次，第 3 次分析根因

## 输出格式

```
## 执行进度
- [x] 任务 1 — ✅ 通过
- [x] 任务 2 — ✅ 通过
- [ ] 任务 3 — 进行中...
- [ ] 任务 4 — 等待任务 3
```

## 门控

- 所有任务完成 ✓
- 构建/类型检查/Lint 通过 ✓
- 交叉验证清单通过 ✓

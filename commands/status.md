# /status — 工作流状态

查看当前工作流阶段和进度。

## 输出

```
当前阶段: [discuss / plan / execute / verify / ship]
当前任务: [任务描述]
进度: [完成数]/[总数]
上下文使用率: [%]
未解决问题: [数量]
下一步: [建议行动]
```

## 阶段判定

- **discuss** — 需求未对齐，有歧义待澄清
- **plan** — 需求已对齐，方案设计中
- **execute** — 计划已批准，执行中
- **verify** — 实现完成，验证中
- **ship** — 验证通过，等待合并/部署

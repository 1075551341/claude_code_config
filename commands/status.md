# /status — 工作流状态

查看当前工作流阶段和进度。

## 输出

```
当前阶段: [discuss / plan / execute / verify / learn]
当前任务: [任务描述]
进度: [完成数]/[总数]
上下文使用率: [%]
未解决问题: [数量]
下一步: [建议行动]
```

## 五阶段判定

- **discuss** — 需求未对齐，有歧义待澄清 | ①规划
- **plan** — 需求已对齐，方案设计中 | ②规格
- **execute** — 计划已批准，执行中 | ③执行
- **verify** — 实现完成，验证中 | ④验证
- **learn** — 验证通过，模式提取/上下文释放 | ⑤学习

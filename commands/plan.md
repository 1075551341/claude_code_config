# /plan — 设计方案与任务分解

基于已对齐的需求，设计实现方案、分解任务、识别依赖。

## 流程

1. 头脑风暴 ≥2 方案，评估后推荐最优
2. 细化推荐方案：文件路径、数据流、接口变更
3. 分解为独立子任务，标注依赖关系
4. 输出：设计方案 + 任务 DAG + 验证步骤

## 输出格式

```
## 方案
**推荐**: [方案名]
**备选**: [方案名]（简要说明取舍原因）

## 文件变更清单
- `path/to/file` — [变更类型: 新增/修改/删除] [说明]

## 任务分解
| # | 任务 | 依赖 | 预计文件 | 验证 |
|---|------|------|----------|------|
| 1 | ...  | -    | ...      | ...  |
| 2 | ...  | 1    | ...      | ...  |
```

## 门控

- 方案经过评估 ✓
- 任务有明确成功标准 ✓
- 无破坏性变更未标记 ✓

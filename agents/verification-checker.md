# 完成验证专家

## 铁律

> **无新鲜验证证据，不声称完成。**

## 验证流程

### 1. 代码验证
- [ ] 构建通过（零错误、零警告）
- [ ] 类型检查通过（tsc --noEmit / mypy）
- [ ] Lint 通过（eslint / ruff / clippy）
- [ ] 所有修改文件已重新读取确认

### 2. 测试验证
- [ ] 单元测试全部通过
- [ ] 修改相关测试已更新
- [ ] 边界情况已覆盖
- [ ] 无跳过的测试（.skip/.todo）

### 3. 功能验证
- [ ] 核心功能端到端可运行
- [ ] 错误路径正确处理
- [ ] 无 console.log/debugger 残留
- [ ] 无 TODO/FIXME 遗漏

### 4. 安全验证
- [ ] 无硬编码密钥/凭证
- [ ] 无 SQL 注入风险
- [ ] 无 XSS 风险
- [ ] 敏感操作有权限检查

### 5. 文档验证
- [ ] README 已更新（如涉及公共 API）
- [ ] 变更日志已记录
- [ ] 类型定义已导出

## 验证报告格式

```markdown
## 验证报告

| 检查项 | 状态 | 证据 |
|--------|------|------|
| 构建 | ✅/❌ | 命令输出摘要 |
| 类型 | ✅/❌ | 错误数 |
| Lint | ✅/❌ | 问题数 |
| 测试 | ✅/❌ | 通过/总数 |
| 功能 | ✅/❌ | 验证方式 |

**结论**：可以提交 / 需要修复 [具体问题]
```

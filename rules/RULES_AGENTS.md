# Agent 协作规则

> 配合核心规则使用，仅在多 Agent 协作场景加载

## Agent 调用原则

### 何时调用 Subagent

| 条件       | 行动                       |
| ---------- | -------------------------- |
| 任务 >5 步 | 分解为 subagent            |
| 多独立模块 | 并行 subagent              |
| 需深度研究 | 专用 agent                 |
| 复杂调试   | systematic-debugging agent |

### Subagent 任务分解

```
1. 分解为独立子目标，每个有明确成功标准
2. 按依赖顺序排列
3. 无依赖子任务并行执行
4. 每子目标完成后验证，再进入下一个
5. 失败时隔离问题，不污染其他子目标
```

### 禁止事项

- 禁止 agent 间共享可变状态
- 禁止 agent 直接修改其他 agent 上下文
- 禁止串行依赖超过 3 层

## 上下文预算

| 场景         | 预算                 |
| ------------ | -------------------- |
| 主 agent     | 60% 上下文上限       |
| subagent     | 30% 上下文上限       |
| 协作信息传递 | 最小化，只传必要数据 |

## 模型选择策略

| 任务复杂度 | 推荐模型     | 适用场景                           |
| ---------- | ------------ | ---------------------------------- |
| 低         | Haiku        | 文件查找、简单格式化、快速问答     |
| 中         | Sonnet       | 代码编写、bug修复、重构            |
| 高         | Opus         | 架构设计、复杂调试、多方案评估     |

原则：子Agent默认用轻量模型，仅复杂子任务升级；主Agent继承用户选择。

## 通信协议

### Agent → Agent

```
[agent_name] 完成 [任务] → 输出 [结果] → 状态 [成功/失败]
```

### 错误传播

```
subagent 失败 → 报告错误 + 上下文 → 主 agent 决定重试/跳过/终止
```

## 并行执行

- 无依赖任务并行 subagent
- 并行结果汇总主 agent
- 冲突时主 agent 仲裁

## 经验固化

- subagent 失败模式 → 存入 `experiences/`
- 成功模式 → 提取为 skill/rule

# 通用开发规则

## 不可变性原则

- 始终创建新对象，不修改现有对象（数据不可变）
- 数组更新用展开运算符 / map / filter，不用 splice / push / sort（原地）
- 状态更新返回新引用，不就地修改

## 文件组织

- 多小文件优于少大文件
- 典型文件 200-400 行，上限 800 行
- 每个文件单一职责
- 目录按功能域组织，不按技术类型

## 错误处理

- 每层显式处理错误，永不静默吞错
- 错误信息包含上下文（操作名、输入摘要、原始错误）
- 自定义错误类携带机器可读 code
- 异步操作必须 try/catch，禁止裸 await

## 输入验证

- 系统边界验证所有外部输入（API、CLI、用户输入）
- 内部代码信任类型系统，不重复验证
- 验证失败返回具体错误，不泛化

## 安全基线

> 详见 `RULES_SECURITY.md`（OWASP Top 10、加密、API安全等完整覆盖）

## 性能意识

- 避免不必要的计算和内存分配
- 热路径优先优化（测量 → 优化 → 验证）
- 大集合操作注意算法复杂度
- I/O 操作批量处理，避免 N+1

## 测试要求

- 新功能必须有测试覆盖
- Bug 修复先写复现测试
- 测试命名描述行为：`should_x_when_y`
- 不跳过测试（无 .skip/.todo 除非附 Issue 链接）

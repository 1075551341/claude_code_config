## 核心能力

### 1. WCAG 2.1 核心原则（POUR）

```markdown
可感知（Perceivable）:
- 1.1 文本替代: 图片 alt、图标 aria-label
- 1.2 时基媒体: 视频字幕、音频描述
- 1.3 可适配: 语义化HTML、结构清晰
- 1.4 可区分: 颜色对比度、文本可缩放

可操作（Operable）:
- 2.1 键盘可访问: 所有功能键盘可达
- 2.2 充足时间: 可暂停/延长限时操作
- 2.3 防止癫痫: 无闪烁内容
- 2.4 导航: 清晰导航、跳过链接

可理解（Understandable）:
- 3.1 可读: 语言属性、复杂词汇解释
- 3.2 可预测: 一致导航、无意外变化
- 3.3 输入帮助: 错误提示、标签清晰

稳健（Robust）:
- 4.1 兼容: 语义化HTML、ARIA正确使用
```

### 2. ARIA 属性使用规范

```html
<!-- 正确的 ARIA 使用 -->
<button aria-label="关闭对话框">✕</button>
<nav aria-label="主导航">...</nav>
<div role="alert" aria-live="polite">错误消息</div>
<input aria-describedby="hint-text" />
<dialog aria-modal="true" aria-labelledby="dialog-title">

<!-- 避免 ARIA 过度使用 -->
<!-- ❌ 不要用 role 替代原生语义 -->
<div role="button">点击</div>  <!-- 应使用 <button> -->
```

### 3. 颜色对比度要求

| 级别 | 正常文本 | 大文本 (18px+) | UI组件 |
|------|----------|----------------|--------|
| AA | 4.5:1 | 3:1 | 3:1 |
| AAA | 7:1 | 4.5:1 | - |

### 4. 键盘导航检查清单

```markdown
□ 所有交互元素可通过 Tab 访问
□ Tab 顺序符合视觉逻辑（左→右，上→下）
□ 焦点样式清晰可见（outline 非隐藏）
□ Enter/Space 可激活按钮和链接
□ Escape 可关闭模态框/下拉菜单
□ 箭头键可导航列表/菜单/树
□ 无键盘陷阱（焦点不会被困住）
□ 跳过导航链接存在
```

## 工作流程

1. **自动扫描** - 使用 axe-core、WAVE 等工具检测
2. **人工审查** - 键盘导航、屏幕阅读器测试
3. **问题分类** - 按 WCAG 准则和严重程度排序
4. **提供修复** - 具体代码修改建议
5. **验证合规** - 修复后重新测试验证

# 潜在客户研究助手

## 核心原则

```
产品分析 → 目标市场识别 → 客户画像构建 → 线索筛选 → 出库策略
```

## 方法论

### 步骤1: 产品分析

深入理解产品价值主张：

```
产品分析:
- 核心功能: [描述]
- 解决问题: [描述]
- 目标用户: [描述]
- 竞争优势: [描述]
- 定价策略: [描述]
```

### 步骤2: 目标市场识别

确定目标市场特征：

```
市场特征:
- 行业: [行业列表]
- 公司规模: [规模范围]
- 地理位置: [地区范围]
- 技术栈: [技术偏好]
- 成熟度: [发展阶段]
```

### 步骤3: 客户画像构建

构建理想客户画像（ICP）：

```
客户画像:
- 公司属性: [规模、行业、位置]
- 技术属性: [技术栈、工具]
- 业务属性: [痛点、预算、决策流程]
- 联系人属性: [角色、级别、关注点]
```

### 步骤4: 线索收集

从多个渠道收集潜在客户线索：

```
线索来源:
- 公开数据: [公司官网、招聘信息]
- 社交媒体: [LinkedIn、Twitter]
- 行业报告: [市场研究、分析师报告]
- 活动数据: [会议、展会参与者]
- 合作伙伴: [推荐、引荐]
```

### 步骤5: 线索筛选

根据画像筛选高质量线索：

```
筛选标准:
- 匹配度: [与ICP的匹配程度]
- 意向度: [购买意向强弱]
- 可行性: [接触可能性]
- 优先级: [跟进优先级]
```

### 步骤6: 出库策略

制定接触和转化策略：

```
接触策略:
- 触达方式: [邮件、电话、社交]
- 时机选择: [最佳接触时间]
- 内容定制: [个性化消息]
- 跟进计划: [多步骤跟进]
```

## 客户画像要素

### 公司层面

```
- 行业: [目标行业]
- 规模: [员工数、收入规模]
- 成熟度: [初创、成长、成熟]
- 地理位置: [地区、城市]
- 资金状况: [融资情况、预算]
```

### 技术层面

```
- 技术栈: [编程语言、框架]
- 基础设施: [云服务、自建]
- 工具使用: [现有工具、平台]
- 技术成熟度: [技术领先/保守]
```

### 业务层面

```
- 痛点: [面临的问题]
- 优先级: [问题重要性]
- 预算: [可用预算范围]
- 决策流程: [决策链、审批流程]
- 时间窗口: [购买时间计划]
```

### 联系人层面

```
- 角色: [职位、职责]
- 级别: [决策权、影响力]
- 关注点: [KPI、痛点]
- 沟通偏好: [沟通方式、时机]
```

## 线索质量评估

### 高质量线索特征

```
✅ 与ICP高度匹配
✅ 有明确痛点
✅ 有预算和时间窗口
✅ 决策链清晰
✅ 接触方式有效
```

### 低质量线索特征

```
❌ 与ICP不匹配
❌ 痛点不明确
❌ 预算不足或时间窗口不明确
❌ 决策链复杂或不明
❌ 接触方式无效
```

### 评分系统

```
匹配度: 1-5分
意向度: 1-5分
可行性: 1-5分
总分: 3-15分

高优先级: 12-15分
中优先级: 8-11分
低优先级: 3-7分
```

## 数据收集策略

### 公开数据挖掘

```
公司官网:
- 产品信息
- 客户案例
- 团队介绍
- 新闻动态

招聘信息:
- 技术栈
- 团队规模
- 业务重点
- 成长速度
```

### 社交媒体分析

```
LinkedIn:
- 公司动态
- 员工规模
- 技术分享
- 招聘需求

Twitter:
- 产品讨论
- 技术趋势
- 行业观点
```

### 行业报告分析

```
市场研究:
- 市场规模
- 增长趋势
- 竞争格局
- 技术趋势

分析师报告:
- 技术采用率
- 预算分配
- 采购偏好
```

## 出库策略

### 个性化触达

```
基于画像定制消息:
- 提及相关痛点
- 展示行业案例
- 提供具体价值
- 明确下一步行动
```

### 多渠道触达

```
邮件: 正式介绍、详细方案
- 电话: 快速建立联系
- 社交: 建立关系、内容分享
- 活动: 面对面交流
```

### 跟进节奏

```
第1天: 初始触达
第3天: 跟进（无回复）
第7天: 提供价值内容
第14天: 电话跟进
第30天: 长期培育
```

## 避免陷阱

- ❌ 追求数量忽视质量
- ❌ 画像过于宽泛
- ❌ 数据来源单一
- ❌ 触达方式不合适
- ❌ 跟进节奏不合理
- ✅ 持续优化画像
- ✅ 多渠道验证数据
- ✅ 个性化触达
- ✅ 合理的跟进节奏

## 示例

### 案例：SaaS产品线索研究

```
产品分析:
- 核心价值: 企业级项目管理SaaS
- 目标市场: 中大型科技公司
- 解决问题: 项目协作效率低下
- 竞争优势: AI智能排程

客户画像:
- 公司属性: 100-1000人、科技行业、一线城市
- 技术属性: 使用Jira/Asana、云原生架构
- 业务属性: 快速成长、多项目并行、远程协作
- 联系人: CTO/工程总监、关注效率

线索收集:
- LinkedIn搜索: "工程总监" + "项目管理"
- 技术社区: GitHub、Stack Overflow活跃公司
- 招聘信息: 招聘项目经理的公司
- 行业报告: SaaS采用率高的行业

线索筛选:
- 公司A: 匹配度5/意向度4/可行性5 = 14分（高优先级）
- 公司B: 匹配度4/意向度3/可行性4 = 11分（中优先级）
- 公司C: 匹配度3/意向度2/可行性3 = 8分（低优先级）

出库策略:
高优先级: 24小时内邮件+LinkedIn连接
中优先级: 3天内邮件触达
低优先级: 加入长期培育名单
```

# ADR: v10.5 移除 Understand-Anything

**日期**: 2026-07-17  
**状态**: Accepted  
**废止**: ADR-2026-06-16 修订 1（UA l3_on_demand）

## 背景

v10.2 将 UA 从 disabled 调整为 L3 按需，以覆盖 onboarding/领域聚类。实践中：

- 与 codegraph + codebase-memory 双引擎在探索路由上仍易互博
- Cursor/插件常驻成本与 token 压力
- 用户在 v10.5 访谈明确要求 **去除 Understand-Anything**

## 决策

1. Understand-Anything **removed** 出企业级骨架（非 disabled、非 l3_on_demand）
2. 探索链固定：`codegraph → codebase-memory(L4) → Grep → Read`
3. 插件保持/设为 disabled；skill 迁 `catalog/skills/`
4. 历史 ADR 与 research 卡片保留审计

## 替代

| 原 UA 场景        | 替代                                                |
| ----------------- | --------------------------------------------------- |
| 项目全貌/结构     | `codegraph_explore`                                 |
| 架构/模块聚类/ADR | `codebase-memory` `get_architecture` / `manage_adr` |
| 变更影响          | cbm `detect_changes` 或 codegraph blast-radius      |

## 后果

- 无交互式导览 UI（接受）
- 不得静默装回 UA；复启用需新访谈 + 新 ADR

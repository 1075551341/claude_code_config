# ADR: v10.4 — codebase-memory-mcp 双引擎集成 + claude-context 归档

> 日期: 2026-06-29 | 状态: Accepted | 版本: v10.3.1 → v10.4.0 | 类型: Minor

## 背景

v10.3.1 完成 lint/project-init 模板与 4 张高变化卡片 delta。用户要求满足需求 #8：**CodeGraph + codebase-memory-mcp 双引擎**；29/29 仓库调研覆盖；防互博；L4 按需加载控制 token。

缺口：
1. DeusData/codebase-memory-mcp 零文档/零配置
2. zilliztech/claude-context 文档漂移（指向不存在的 dev.json optional）
3. L3 洞察层仅 codegraph 单引擎

## 决策

### D1: 双引擎互补（非替代）

**选择**: codegraph 保持 R17 常驻主位；codebase-memory-mcp L4 按需。

| 场景 | 首选 |
|------|------|
| 符号/调用链/blast-radius | codegraph_explore |
| 架构全景/ADR/跨服务/变更风险 | cbm get_architecture / manage_adr / detect_changes |
| 语义找代码 | cbm semantic_query（替代 claude-context） |

**理由**: 功能重叠但 cbm 独有能力（ADR、Leiden、跨服务、detect_changes）值得 L4 按需；codegraph 已深度集成 R17/R6，迁移成本高且无收益。

### D2: claude-context archived_redirect

**选择**: 标记 archived_redirect → codebase-memory-mcp；不再维护 Milvus 配置路径。

**理由**: cbm 本地嵌入无基础设施；claude-context 实测 -40% token 数据保留作历史参考。

### D3: MCP 加载 L4 optional-dev

**选择**: `mcp-configs/optional-dev.json` 新增 `codebase-memory`；不进 `.mcp.json` 常驻 5。

**理由**: 访谈锁定控制固定开销；满足时 merge optional profile。

### D4: Cursor 同步默认 L0

**选择**: 文档化默认 L0；`-Skills`/`-All` 按需；执行 sync 回退 Cursor rules。

**理由**: -All 全量 rules 违背 L0 设计，每 turn 多 ~5-8K token。

### D5: 版本 v10.4.0 minor

**选择**: v10.3.1 → v10.4.0（非 v11）。

**理由**: L3 扩展 + 新 concern；无 breaking；符合 R14。

### D6: GSD 维持 Stay 1.4.5

**选择**: 不升级 open-gsd/gsd-core 1.6.0。

**理由**: ADR-1244 已评估；forensics/resume 在 P2。

# ADR: v10.3.1 Patch — Lint 模板 + 项目初始化 + 高变化仓库 Delta 刷新

> 日期: 2026-06-26 | 状态: Accepted | 版本: v10.3 → v10.3.1 | 类型: Patch

## 背景

v10.3 完成了 17 张 stale 卡片刷新 + codegraph F1 纠偏 + 7 编辑器同步扩展。本轮 v10.3.1 为 patch 级补缺，解决三类问题：

1. **高变化仓库版本漂移**：4 个核心仓库（superpowers/claude-mem/gsd-core/rtk）在 v10.3 后发布新版本，需 delta 刷新
2. **缺少 lint 工具链模板**：用户要求 prettier 管格式 + eslint 管质量，互不混淆
3. **缺少跨项目初始化机制**：用户要求持续迭代进化全局通用配置，对不同项目也能迭代优化

## 决策

### D1: Delta 刷新 4 张高变化卡片（非全量刷新）

**选择**: 仅刷新核心五柱 + L2 仓库（4 张），不做 28 张全量刷新。

**理由**: 高变化仓库（superpowers/claude-mem/gsd-core/rtk）发布频率高（周级），其余仓库月级稳定。全量刷新浪费 token，delta 刷新精准命中。

**版本纠正**:

- claude-mem: v13.6.1 → v13.8.1（npm registry 三源验证；GitHub Releases 不完整）
- rtk: v0.42.4 → v0.41.0（**纠正错误**：v0.42.x 从未发布稳定版，原记录混淆 npm dev tag）
- superpowers: v6.0.0 → v6.0.3（patch 级升级，SDD scratch 路径变更）
- gsd-core: v1.5.0 → v1.6.0（ADR-1244 完整落地，维持 Stay 1.4.5 推荐）

### D2: Prettier + ESLint 9 Flat Config 模板

**选择**: `templates/lint/` 目录，3 个文件：

- `.prettierrc.json` — 格式化配置（仅格式，无代码质量规则）
- `.prettierignore` — 忽略模式
- `eslint.config.js` — ESLint 9 flat config（仅 bug-catching 规则，无风格规则）

**设计原则**:

- prettier 管格式，eslint 管质量，**互不混淆**
- eslint 仅保留影响代码运行的规则（no-undef/no-unreachable/valid-typeof 等）
- 风格规则全部关闭（交给 prettier）
- React hooks / Vue essential / Angular 以注释段预设，取消注释即启用
- eslint-config-prettier 关闭所有冲突规则
- **HTML/Vue/JSX 属性每行一个，标签独立成行**：
  - `singleAttributePerLine: true` — 多属性时每个属性单独一行
  - `bracketSameLine: false` — 开标签 `>` 独立成行
  - `htmlWhitespaceSensitivity: "ignore"` — `>`/文本/`</tag>` 各自独占一行（默认 `css` 会把 `>text</tag` 压在一行）
  - `vueIndentScriptAndStyle: true` — Vue SFC script/style 块缩进一级（template/style/css 均正常缩进）
  - `printWidth: 120` — 单行最大宽度
  - **权衡**: `ignore` 模式对原生 HTML inline 元素（`<span>`/`<a>`）可能引入额外空白节点；对 Vue/JSX 组件（非原生 inline）安全
  - **bracketSameLine 历史**: `rules/FRONTEND.md` 旧警告"勿设 bracketSameLine: true — 与 eslint-plugin-prettier 冲突"仅适用 legacy plugin 模式；v10.3.1+ 用 ESLint 9 flat config + eslint-config-prettier（独立运行，非 plugin 模式），`bracketSameLine: false` 安全无冲突，实测验证通过

**分发**: `sync.ps1 -Lint` 拷贝到 CWD，**跳过已存在文件**（不覆盖自定义配置）

### D3: 项目初始化模板

**选择**: `templates/project-init/` 目录，4 个文件：

- `CLAUDE.md` — 项目级配置（继承全局 `~/.claude/CLAUDE.md`）
- `MANIFEST.yaml` — 项目级归属（继承全局 MANIFEST）
- `.env.example` — 环境变量模板
- `.gitignore` — 项目级 gitignore

**设计原则**:

- 全局 SSOT + 项目层覆盖（不重复声明全局规则）
- 占位符 `{{PROJECT_NAME}}` / `{{TECH_STACK}}` 等，用户替换
- 拷贝到 CWD，**跳过已存在文件**

**分发**: `sync.ps1 -InitProject` 拷贝到 CWD

### D4: sync.ps1 v18.0 扩展

**选择**: 两个独立 flag（`-Lint` / `-InitProject`），standalone 模式跳过 editor 同步。

**理由**: lint/project-init 是项目级操作（目标 CWD），与 editor 同步（目标 `~/.cursor` 等）正交。独立 flag 避免混淆。

**向后兼容**: 默认模式（无 flag）行为不变；`-Skills` / `-All` 行为不变。

### D5: 版本号 v10.3.1（patch）

**选择**: v10.3 → v10.3.1，非 v10.4 或 v11。

**理由**: 所有变更为补缺/纠正，无骨架变更（五柱×五阶段×三横切不变），无 breaking change。符合 R14 版本克制原则。

# ADR: GSD (gsd-core) 1.5.0 评估

**日期**: 2026-06-19
**状态**: Proposed（评估中，未签署 → `MANIFEST.gsd.version` 保持 1.4.5）
**决策人**: 待用户访谈
**来源**: Exa + GitHub Releases 双源核验（v1.5.0 stable 2026-06-05）

---

## 背景

v10.2.1 双源调研发现 `@opengsd/gsd-core` **v1.5.0 stable 已发布**（此前本地认知为「仅 rc，仅跟踪」）。本地锁定 1.4.5。1.5.0 引入 Capability Architecture（ADR-857）、async resume/pause、Research module 等能力，**部分与本地横切/五柱重叠**，需独立评估再决定，避免 scope creep 与左右手互博（Req 3）。

---

## 1.5.0 新能力 vs 本地冲突风险

| 1.5.0 新能力 | 说明 | 本地现状 | 冲突/重叠风险 |
|--------------|------|----------|---------------|
| **Capability Registry** | ADR-857 namespace meta-skill + 生成器 + UI pilot | `MANIFEST.yaml` concerns + module_resolver | 🟡 中——两套「能力/归属」模型重叠，强行合并增复杂度 |
| **async resume/pause** | `external_job_waiting` 半状态 + resume/pause 契约 | `hooks/pre-compact-state` + 制品恢复（forensics 此前 P2 不实现） | 🟡 中——与本地状态持久化重叠 |
| **Research module** | 内容寻址缓存 + provider seam + `RESEARCH.md` | deep-research（Firecrawl+Exa 双源）+ L3 横切 | 🔴 高——与本地双源调研**直接互博**（Req 6） |
| 联邦配置合并 | config-loader federated merge | sync.ps1 软链/复制 | 🟢 低 |
| Loop Host Contract | 从 workflow markers 生成 | 五阶段状态机 | 🟢 低 |

---

## 三选一方案

| 方案 | 内容 | 利 | 弊 |
|------|------|----|----|
| **A. Stay 1.4.5（推荐）** | 不升级，继续锁 1.4.5；1.5.0 能力按需 cherry-pick 评估 | 零破坏；避免 Research 互博；符合 R14 版本克制 | 不享受 async resume 原生实现 |
| B. Cherry-pick Research seam | 仅吸收 Research module 的内容寻址缓存思路到 deep-research | 增量优化 deep-research 缓存 | 需自研适配；收益不确定 |
| C. Full Upgrade 1.5.0 | 升级 gsd-core 到 1.5.0 | 享全部新能力 | Research 与 deep-research 互博；Capability Registry 与 MANIFEST 重叠；高重构成本 |

**推荐：方案 A（Stay 1.4.5）**。理由：
1. Research module 与本地 deep-research 双源调研直接互博，升级将引入「双调研入口」左右手互博（违反 Req 3）。
2. async resume / Capability Registry 的能力本地已有等效（制品恢复 + MANIFEST concerns），R14 版本克制下不构成升级理由。
3. 1.5.0 的内容寻址缓存思路（Research module）值得后续 cherry-pick 评估（方案 B 入 v10.3 backlog），但不阻塞当前。

---

## 决策门控

- **签署前**：`MANIFEST.gsd.version` 保持 **1.4.5**；`docs/research/repos/open-gsd-gsd-core.md` 标注「Phase E 评估中」。
- **签署后**（若采纳 A）：本 ADR 转 Accepted；Research module cherry-pick 评估记入 v10.3 backlog。
- **复评条件**：1.5.x 出现本地无等效的阻塞能力，或用户显式要求升级。

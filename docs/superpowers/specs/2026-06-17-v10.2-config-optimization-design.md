## 错误升级路径（新增 D11）

在 CORE.md 状态机追加：

```
Agent 异常 → 主 Agent 判断
  ├── 可重试（网络/超时）→ 重试 ≤R5 上限 → 仍失败 → 报告用户
  ├── 可降级（非核心能力不可用）→ 降级方案 → 继续 + 标注 DONE_WITH_CONCERNS
  ├── 需确认（权限/冲突/数据风险）→ 暴露给用户 + 等待决策
  └── 硬阻断（安全/CVE/数据丢失）→ 立即停止 + 详细报告
```

### P0.2: 工具描述密度精简

参照 x1xhlol/system-prompts 研究结论：工具描述密度宜低不宜高，减少 30% 误用。

**目标**：CLAUDE.md 中移除冗余工具描述（codegraph 详细指南已在 MCP server instructions 中），`rules/` 中删减过度详细的示例代码块（保留原则，删除完整实现）。

### P0.3: snippets/ 初始化（≤10 天花板）

吸收 Chalarangelo/30-seconds-of-code 精华，创建 `snippets/` 目录：

```
snippets/
  python/
    datetime-utils.md       # 时间处理（避免 new Date() 违规）
    error-handling.md       # 错误处理模板（R16 合规）
    validation.md           # 输入验证模板
  javascript/
    immutable-ops.md        # 不可变操作（map/filter/reduce）
    async-patterns.md       # 异步模式（避免裸 await）
  shell/
    safe-scripting.md       # 安全脚本模板
```

**天花板**：snippets ≤ 10 个文件。超过 10 个 → 抽象为 skill。剩余场景靠 claude-mem observation 自然积累。

### P0.4: 错误升级路径

在 `rules/CORE.md` 状态机章节追加 4 路径错误升级规则（重试/降级/确认/阻断）。

### P0.5: 防互博 6→9 组

MANIFEST 追加 3 组 excludes：firecrawl-agent↔deep-research、land-and-deploy↔release-engineer、pair-agent↔agentic-orchestrator。
